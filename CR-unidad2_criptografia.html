1. APARTADO TEÓRICO: Fundamentos Matemáticos y Evolución

Base Matemática

Aritmética Modular y Números Primos: La criptografía moderna, especialmente la asimétrica (RSA), recae en la dificultad computacional de factorizar el producto de dos números primos grandes (n = p * q). La aritmética modular asegura que las operaciones matemáticas se mantengan dentro de un límite definido, formando "relojes" criptográficos.

Cifrado Simétrico: Se apoya en procesos de confusión y difusión (Redes de Feistel o Sustitución-Permutación).

Evolución y Contexto de Seguridad

  • Clásica (César, Vigenère): Vulnerable al análisis de frecuencias. Sirvió de base conceptual.
  • DES (Data Encryption Standard): Estándar de 56 bits. Vulnerable hoy por fuerza bruta debido a su longitud de clave corta.
  • AES (Advanced Encryption Standard): Estándar actual (128/192/256 bits). Matemáticamente seguro, opera en matrices de estado (Rijndael).
  • RSA: Pilar de la infraestructura de clave pública (PKI). Seguro si las claves son >= 2048 bits, pero amenazado a futuro por la computación cuántica (Algoritmo de Shor).

2. MÉTODOS Y LABS: Implementación Algorítmica y Despliegue Técnico

A continuación, el flujo matemático para generar claves RSA y el despliegue equivalente en terminal.

Matemática Subyacente: Cálculo de Claves RSA

1. Elegir primos grandes p y q. Calcular n = p * q.

2. Calcular la función de Euler: φ(n) = (p-1) * (q-1).

3. Elegir exponente público e tal que el MCD(e, φ(n)) = 1 (Algoritmo de Euclides).

4. Calcular clave privada d donde (d * e) mod φ(n) = 1 (Algoritmo de Euclides Extendido).

Comandos Reproducibles (OpenSSL)

# 1. Generar clave privada RSA (2048 bits)
openssl genrsa -out privada.pem 2048

# 2. Extraer la clave pública correspondiente
openssl rsa -in privada.pem -pubout -out publica.pem

# 3. Cifrar archivo simétricamente con AES-256-CBC (Ejemplo para grandes volúmenes de datos)
openssl enc -aes-256-cbc -salt -in secreto.txt -out secreto.enc

# 4. Generar Hash SHA-256 para validación de integridad
openssl dgst -sha256 archivo.pdf

3. FICHAS RÁPIDAS: Tarjetas de Memorización

Pasa el ratón (o pulsa) sobre las tarjetas para ver la definición y el ejemplo.

Clave Privada vs Pública
Definición: Par matemático donde una cifra (pública) y la otra descifra (privada).

Ejemplo: Cualquiera puede enviarte un email cifrado con tu clave pública, pero solo tú lo lees con tu privada.
Salting
Definición: Añadir datos aleatorios a una contraseña antes de hacer su hash para evitar ataques de diccionarios/rainbow tables.

Ejemplo: Hash("password123" + "x8F9s").
Vector de Inicialización (IV)
Definición: Bloque de bits aleatorios usado para asegurar que cifrar el mismo texto dos veces genere cifrados distintos (ej. en modo CBC).
Nonce
Definición: "Number Used Once". Número arbitrario usado solo una vez en una comunicación para prevenir ataques de repetición.
Teorema de Fermat
Definición: Base para test de primalidad. a^(p-1) ≡ 1 (mod p).

Aplicación: Vital para encontrar de manera rápida los grandes números primos p y q que sustentan RSA.

4. CASOS PRÁCTICOS: Soluciones Oficiales Integradas

Caso Práctico 3: Cifrado Simétrico (DES)

Contexto: Alice y Bob desean comunicarse usando un cifrado simétrico (DES).

Solución Oficial de Plataforma:

  • Generación de Clave: Acuerdan de forma segura una clave secreta de 56 bits (64 bits total, 8 de paridad).
  • Cifrado (Alice): Divide el texto plano en bloques de 64 bits. Aplica el algoritmo DES a cada bloque usando la clave secreta (permutaciones, sustituciones y funciones de Feistel). Envía el texto cifrado.
  • Descifrado (Bob): Usa la misma clave secreta de 56 bits. Al ser simétrico, aplica el proceso inverso para recuperar el original.
  • Confidencialidad: Radica totalmente en mantener en secreto la clave compartida. Nota: Hoy en día DES se considera inseguro por su clave corta.
Caso Práctico 4: Cifrado Asimétrico (RSA)

Contexto: Alice quiere enviar un mensaje a Bob garantizando confidencialidad con RSA.

Solución Oficial de Plataforma:

  • Generación de Claves (Bob): Bob genera clave pública (e, n) y privada (d, n). Distribuye la pública.
  • Cifrado (Alice): Convierte su mensaje a formato numérico M. Lo cifra utilizando la clave pública de Bob con la fórmula: C = Me mod n. Envía C.
  • Descifrado (Bob): Recibe el criptograma C y aplica su clave privada con la fórmula: M = Cd mod n.
  • Confidencialidad: Solo Bob puede descifrarlo porque solo él posee el exponente privado d. Sin este, es computacionalmente inviable obtener el mensaje.

5. GLOSARIO TÉCNICO: Terminología Específica

Cifrado Simétrico / Asimétrico
Simétrico: Misma clave para cifrar/descifrar (AES). Muy rápido, ideal para volumen. Asimétrico: Par de claves Pública/Privada (RSA). Lento, ideal para intercambio de claves de sesión o firmas.
Colisión Hash
Evento donde dos entradas de datos distintas producen exactamente el mismo resumen o salida Hash (ej. debilidad de MD5). Una buena función criptográfica debe ser resistente a colisiones.
Entropía
Medida de la aleatoriedad y la imprevisibilidad en los datos. Los generadores de claves (PRNG) necesitan alta entropía del sistema (movimientos de ratón, ruido térmico) para ser seguros.
Padding (Relleno)
Datos adicionales añadidos al final del texto plano para asegurar que cumple con el tamaño de bloque exacto requerido por algoritmos simétricos como AES o DES.
Key Exchange (Intercambio de Claves)
Mecanismo (como Diffie-Hellman o intercambio vía RSA) por el cual dos partes establecen una clave simétrica compartida de manera segura sobre un canal inseguro.

Alternativas de validación de consenso (Contexto Blockchain/Criptografía Descentralizada):

  • PoW (Proof of Work): Alta seguridad basada en poder computacional matemático. Control de seguridad: Resistencia extrema, pero alto coste energético.
  • PoS (Proof of Stake): Eficiencia energética basada en posesión de activos. Control de seguridad: Reducción del ataque del 51% mediante penalización económica (Slashing).
