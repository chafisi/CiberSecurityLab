Codificación del canal

Garantizar confiabilidad

Definición: Añadir redundancia estructurada para que el receptor detecte o corrija errores del canal.
Ejemplo: Forward Error Correction (FEC) en transmisiones de video.

Modelo de Shannon

Definición: Teorema que establece la tasa máxima de transferencia de datos libre de errores. \( C = B \log_2(1 + S/N) \)
Ejemplo: Determinar el límite teórico de una conexión de fibra óptica.

Técnicas de detección de errores

Bits de paridad (ECC RAM)

Definición: Añade un bit para que el número de '1's sea par/impar.
Caso: Prevención de corrupción en memoria volátil de servidores.

Suma de comprobación (Checksums)

Definición: Suma aritmética de bloques de datos que se envía junto al mensaje.
Caso: Verificación de integridad en cabeceras IP/TCP.

CRC (Cyclic Redundancy Check)

Definición: División polinómica que genera un resto utilizado como firma.
Caso: Detección de alteraciones en tramas Ethernet e IoT.

Técnicas de corrección de errores

Códigos de bloque

  • Hamming: Corrige errores de 1 bit.
  • Reed-Solomon: Excelente para ráfagas de errores (Discos ópticos/RAID).

Códigos convolucionales

  • Viterbi / GPS: Procesan bits de forma continua, ideal para señales satelitales con bajo SNR.

Códigos avanzados

  • LDPC: Aproximación al límite de Shannon (Wi-Fi 6 / 5G).
  • Turbo Codes: Iterativos, usados en LTE.

Aplicaciones en seguridad

Garantía de integridad y disponibilidad

Asegura que los sistemas críticos no caigan ante pequeñas corrupciones lógicas.

Protección ante interferencias deliberadas

Uso de redundancia agresiva para combatir tácticas de jamming electromagnético.

Laboratorio: El Pipeline Seguro (Relación con cifrado)

El orden correcto es fundamental. Selecciona las fases para ver el resultado.

1. Compresión de fuente: Elimina patrones (Reduce predictibilidad para criptoanálisis).
2. Cifrado: Transforma la data en seudoaleatoria (Garantiza confidencialidad).
3. Codificación de canal: Añade redundancia matemática (Garantiza integridad ante ruido físico).
✅ Éxito: Si ciframos antes de comprimir, la compresión es imposible (el cifrado elimina la redundancia aparente).

Limitaciones y Desafíos

Sobrecarga de procesamiento

Algoritmos avanzados como LDPC exigen hardware dedicado (ASICs/FPGAs) para mantener el Throughput.

Latencia y consumo energético

A mayor corrección (Turbo Codes), mayor consumo de batería, crítico en dispositivos IoT.

Laboratorio: Amenazas cuánticas

Ruptura de algoritmos clásicos

Algoritmo de Shor compromete RSA/ECC, afectando la fase de cifrado del canal.

Interferencia cuántica controlada

Manipulación de estados superpuestos para alterar silenciosamente tramas codificadas.

Códigos post-cuánticos

Transición a criptografía basada en retículos (Lattices) inmune a computación cuántica.
