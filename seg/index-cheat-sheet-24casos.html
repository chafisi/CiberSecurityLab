Esquema Mental Táctico: 24 Casos

Requisitos en línea, Flujos, Validación y Glosarios por Bloque

🚨 BLOQUE A: Amenazas Base y Gestión de Incidentes

1. Ransomware (Hospital)

  • RFMonitorizar, aislar red, restaurar backups.
  • RNFRTO < 4h, disponibilidad clínica (99%).
  • TEDR, segmentación L2/L3, backups inmutables.
  • LRGPD (datos salud), Notificación brechas (72h).
🔄 Flujo: Detecta cifrado ➔ EDR aísla ➔ Activa SOC ➔ Restaura Backup ➔ (Exc: Continuidad manual)
✅ VAL: Simulacro Phishing, Test Restore.

2. Credential Stuffing

  • RFExigir MFA, bloquear cuentas tras N fallos.
  • RNFAuth < 2s, alta escalabilidad portal web.
  • TArgon2/bcrypt, análisis heurístico, WAF.
  • LPSD2 (Banca), RGPD (Integridad).
🔄 Flujo: Usuario loguea ➔ Pide MFA ➔ Valida ➔ (Exc: Bloqueo por fuerza bruta)
✅ VAL: Hydra/Burp (Fuerza bruta).

20. 0-Day (Log4Shell)

  • RFEvaluar riesgo, inventariar dependencias, parchear.
  • RNFReducir MTTR crítico, gobernanza fluida.
  • TSCA, SAST, monitorización CVSS, IPS.
  • LNIST RMF, DORA (Resiliencia financiera).
🔄 Flujo: Detecta CVE ➔ Identifica activos ➔ Parchea/Aísla ➔ (Exc: Mitigación temporal sin parche)
✅ VAL: Escáner SCA, Auditoría riesgo residual.
📖 Glosario Bloque A: RTO: Recovery Time Objective. EDR: Endpoint Detection & Response. SOC: Security Operations Center. MFA: Multi-Factor Authentication. MTTR: Mean Time To Respond/Repair. SCA: Software Composition Analysis. SAST: Static App Sec Testing. CVE/CVSS: Common Vulnerabilities & Exposures / Scoring System. DORA: Digital Operational Resilience Act.
🔑 BLOQUE B: Criptografía, Identidad y PKI

3. Transición DES a AES

  • RFUsar algos vigentes, auditar longitudes.
  • RNFRendimiento sin latencia, compatibilidad.
  • TOpenSSL, AES-256, SHA-256.
  • LEsquema Nacional Seguridad (ENS), NIST.
🔄 Flujo: Detecta obsoleto ➔ Migra a AES ➔ (Exc: Fallo migración -> Restore)
✅ VAL: testssl.sh (Análisis cripto).

4. Certificados Falsos

  • RFValidar estado cert, generar alertas.
  • RNFBaja latencia en verificación (OCSP).
  • TCert Transparency (CT), OCSP, CRL.
  • LeIDAS (Firmas y certs europeos).
🔄 Flujo: Navegador recibe Cert ➔ Valida OCSP/CT ➔ (Exc: Revocado -> Bloquea)
✅ VAL: Monitorización CT Logs.

13. Certs. Caducados

  • RFSupervisar caducidad, renovar auto.
  • RNFZero-downtime, confiabilidad total.
  • TACME (Let's Encrypt), PKI, Scripts.
  • LRGPD (Cifrado en tránsito), PCI-DSS.
🔄 Flujo: Detecta expiración ➔ ACME renueva ➔ Despliega ➔ (Exc: Fallo -> Alerta manual)
✅ VAL: Qualys SSL Labs.

14. MFA Robusto FIDO2

  • RFAutenticar FIDO2, registro seguro.
  • RNFUsabilidad alta, multi-dispositivo.
  • TWebAuthn, Hardware Keys, Cripto Asim.
  • LPSD2, RGPD (Biometría si aplica).
🔄 Flujo: Loguea ➔ Pide Llave HW ➔ Firma Cripto ➔ (Exc: Llave perdida -> Revoca)
✅ VAL: Simulacros Phishing MitM.
📖 Glosario Bloque B: AES/DES: Advanced / Data Encryption Standard. PKI: Public Key Infrastructure. CRL/OCSP: Cert Revocation List / Online Cert Status Protocol. CT: Certificate Transparency. ACME: Auto Cert Management Environment. FIDO2/WebAuthn: Fast IDentity Online / API web de Auth. eIDAS: Regulación EU Identidad/Firmas Electrónicas.
🧱 BLOQUE C: Seguridad Perimetral y Redes (L2-L4)

5. Errores Firewall

  • RFValidar reglas previas, registro cambios.
  • RNFDisponibilidad 99.9%, interfaz segura.
  • TSegmentación, Staging, MFA Admin.
  • LISO 27001 (Control accesos/cambios).
🔄 Flujo: Propone regla ➔ Revisa/Prueba ➔ Despliega ➔ (Exc: Fallo -> Rollback)
✅ VAL: Nmap, Config Drift Detection.

6. Migración a NGFW

  • RFInspección DPI, bloquear evasivas (Tor).
  • RNFRendimiento sin latencia DPI, escalable.
  • TApp-ID, descifrado SSL, Integración SIEM.
  • LRGPD (Auditoría de navegación corporativa).
🔄 Flujo: Usuario lanza VPN ➔ NGFW DPI detecta ➔ Bloquea ➔ (Exc: Falso Positivo)
✅ VAL: Test Evasión (Pentest), Tuning SOC.

7. Snort IDS (Uni)

  • RFDetectar firmas/escaneos, generar alertas.
  • RNFNo perder paquetes (wire-speed).
  • TPuerto SPAN/TAP, Preprocessors, Syslog.
  • LENS (Monitorización contínua).
🔄 Flujo: Snort lee tráfico ➔ Detecta SYN ➔ Alerta SIEM ➔ (Exc: Tráfico IT legítimo)
✅ VAL: Escaneo con Nmap/Masscan.

8. IPS Financiero

  • RFBloqueo real-time, modo alerta previo.
  • RNFBaja tasa FP, alta estabilidad.
  • THeurística, Despliegue gradual.
  • LPCI-DSS (Protección datos tarjeta).
🔄 Flujo: Regla en modo alerta ➔ Evalúa FP ➔ Pasa a Bloqueo ➔ (Exc: Corta servicio -> Rollback)
✅ VAL: Traffic Replay, Tuning iterativo.

9. ARP Spoofing

  • RFVerificar IP-MAC, bloquear suplantación.
  • RNFLatencia nula en Capa 2.
  • TDynamic ARP Inspection (DAI), DHCP Snooping.
  • LRGPD (Prevenir fuga de datos locales).
🔄 Flujo: Switch lee ARP ➔ Valida contra tabla ➔ Inconsistencia -> Drop.
✅ VAL: Ataque con Ettercap/arpspoof.

10. IPSec VPN

  • RFTúnel cifrado, renegociar auto claves.
  • RNFEstabilidad de enlace, failover.
  • TIKEv2, AES-GCM, SHA-2, PFS (Forward Secrecy).
  • LRGPD (Transferencia de datos segura).
🔄 Flujo: GW Negocia IKEv2 ➔ Autentica ➔ Abre túnel ESP ➔ (Exc: Falla Auth -> Cierra)
✅ VAL: Wireshark (Comprobar payload cifrado).
📖 Glosario Bloque C: DPI: Deep Packet Inspection. NGFW: Next-Gen Firewall. FP: Falso Positivo. IDS/IPS: Intrusion Detection / Prevention System. DAI: Dynamic ARP Inspection. IKEv2 / ESP: Internet Key Exchange / Encapsulating Sec Payload (IPSec). PFS: Perfect Forward Secrecy.
🌐 BLOQUE D: AppSec y Transmisión de Información (L7)

11. Equifax (Parches)

  • RFFlujo formal parcheo, detectar dependencias.
  • RNFMinimizar MTTP, cero regresiones.
  • TSCA, Nessus, Pipeline CI/CD, Staging.
  • LRGPD, PCI-DSS (Cumplimiento parches).
🔄 Flujo: Detecta vulne ➔ Prueba Staging ➔ CI/CD Despliega ➔ (Exc: Fallo -> Mitigación WAF)
✅ VAL: KPIs de Parcheo (MTTP), Pentest.

12. Robo vía XSS

  • RFSanear inputs, codificar salidas, proteger session.
  • RNFRendimiento fluido, Mínimos FP.
  • TCSP, Cookies HttpOnly/Secure, SAST.
  • LPSD2, RGPD (Token = Dato personal).
🔄 Flujo: Introduce dato ➔ App Sanitiza/Codifica ➔ Emite HTTP ➔ (Exc: Payload -> Drop/Alerta)
✅ VAL: BurpSuite (DAST XSS), ZAP.

15. Huffman (JPEG)

  • RFComprimir antes de cifrar, limpiar metadata.
  • RNFCalidad perceptual, baja latencia.
  • TJPEG Huffman, TLS 1.3, Padding uniforme.
  • LRGPD (Privacidad por Diseño).
🔄 Flujo: Sube img ➔ Quita EXIF ➔ Comprime ➔ Cifra TLS ➔ (Exc: Oráculo longitud -> Pone Padding)
✅ VAL: Análisis Canal Lateral (Tamaño Ciphertext).

16. CRIME (Compresión)

  • RFSeparar secretos de inputs, NO co-comprimir.
  • RNFRendimiento sin TLS compression.
  • THPACK context isolation, Padding Random.
  • LISO 27001, RGPD (Evitar fuga cookies).
🔄 Flujo: Req HTTP/2 ➔ Aísla cookies ➔ Cifra sin compresión TLS ➔ (Exc: HTTP/2 inseguro -> Padding)
✅ VAL: Escáner de Vulnerabilidad Oráculo.

17. Reed-Solomon

  • RFDetectar/Corregir bits corruptos, loguear BER.
  • RNFFlujo lectura continuo, sin paradas.
  • TAlgoritmo RS (n, k), Buffering óptico.
  • LENS/Leyes archivo (Retención íntegra a largo plazo).
🔄 Flujo: Lee CD/DVD ➔ Detecta daño físico ➔ RS Reconstruye bytes ➔ (Exc: Daño masivo -> Ilegible)
✅ VAL: Test físico (Discos dañados controlados).

18. LDPC (Wi-Fi/5G)

  • RFCorregir errores bajo ruido, detectar Jamming.
  • RNFBaja latencia proceso, resiliencia alta SNR.
  • TBelief Propagation, Iteraciones LDPC, WPA3.
  • LNormativas de telecomunicaciones (Disponibilidad).
🔄 Flujo: Detecta ruido ➔ Itera LDPC ➔ Reconstruye señal ➔ (Exc: Jamming extremo -> Handover)
✅ VAL: Generador de Ruido SNR Lab.
📖 Glosario Bloque D: CI/CD: Cont. Integration / Deployment. MTTP: Mean Time To Patch. XSS: Cross-Site Scripting. CSP: Content Security Policy. HPACK: HTTP/2 Header Compression. RS/LDPC: Reed-Solomon / Low-Density Parity-Check. SNR/BER: Signal-to-Noise Ratio / Bit Error Rate.

✅ VAL: Revisión EIPD, Verificación HW Audio.

22. Europol (Cooperación)

  • RFPreservar datos rápido, cooperar internacional.
  • RNFRapidez 24/7 (Punto de contacto).
  • THashing (SHA-2), Sellos Temporales, Custodia.
  • LConvenio Budapest, MLAT, LOPD.

✅ VAL: Cadena Custodia, Trazabilidad forense.

23. AEPD (Consentimiento)

  • RFConsentimiento granular explícito, facilitar ARSOPL.
  • RNFTransparencia, lenguaje claro.
  • TLogs Inmutables Consentimiento, CRM Seguro.
  • LRGPD / LOPDGDD España.
🔄 Flujo: Muestra Banner ➔ Usuario Elige ➔ Loguea timestamp ➔ (Exc: Usuario retira -> Cesa uso)
✅ VAL: Simulacro ARSOPL, Auditoría Base Legal.

24. CNIL (Dark Patterns)

  • RFBotón "Rechazar todo" equivalente a "Aceptar".
  • RNFAusencia de manipulación visual UX/UI.

  • LRGPD, ePrivacy (Francia).

✅ VAL: Test Usabilidad (UX), Auto-scanner Cookies.
📖 Glosario Bloque E: GRC: Governance, Risk & Compliance. SGSI: Sistema Gestión Seg. Info. PDCA: Plan-Do-Check-Act. APDA/AEPD/CNIL: Agencias de Protección de Datos (Andorra/España/Francia). 🧠 HACKEANDO EL EXAMEN: Despliegue de Patrones de Respuesta

No memorices cada caso desde cero; memoriza la plantilla de respuesta. Si identificas el bloque de la pregunta, el 80% se autocompleta con estas reglas universales:

🧩 Leyes Universales de Clasificación de Requisitos

RF
Funcionales: Qué debe hacer exactamente el sistema. Truco: Invierte el ataque. Si el ataque roba -> RF es "Cifrar/MFA". Si paraliza -> RF es "Aislar/Restaurar". Usa siempre verbos de acción directa.
RNF
No Funcionales: Cómo de bien debe hacerlo (Métricas/Calidad). Truco: Atacan al negocio. Usa siempre "Alta Disponibilidad (>99%)", "Latencia nula/Wire-speed", "Usabilidad transparente", "Tiempos mínimos (RTO/MTTR)".
T
Técnicos: Tecnologías, algoritmos y protocolos reales. Truco: Aquí van las siglas duras (AES, WAF, EDR, SIEM, IKEv2). Si dudas en monitorización, pon "Integración con SIEM y Syslog" (aplica casi siempre).
L
Legales: Marco normativo y compliance. Truco: "RGPD" para datos personales/europeos. "ENS / ISO 27001" para procesos y auditorías. "PSD2 / DORA" si lees la palabra 'Finanzas/Banca/Pagos'.

🛠️ Modelos por Bloque Temático

🚨 A: Incidentes Base

Técnicas (Requisito Técnico - T):
  • EDR y Segmentación L2/L3 (Caso 1)
  • Backups Inmutables (Caso 1)
  • MFA y Hashes Argon2 (Caso 2)
  • SCA / SAST / CVSS (Caso 20)
Flujo Táctico:
[ Detecta Ataque / CVE ]
[ Aísla Red / Contiene ]
[ Mitiga / Parchea / Bloquea ]
[ Restaura Servicio Operativo ]
[ Exc: Plan Manual / Bypass ]
Validación:
  • Simulacros de Phishing
  • Pruebas de Restauración (Restore)
  • Escáneres Automatizados (SCA)

🔑 B: Cripto y PKI

Técnicas (Requisito Técnico - T):
  • AES-256 / OpenSSL (Caso 3)
  • CT Logs / OCSP / CRL (Caso 4)
  • ACME / Let's Encrypt (Caso 13)
  • FIDO2 / WebAuthn (Caso 14)
Flujo Táctico:
[ Inicia Conexión / Petición ]
[ Valida Identidad / Certificado ]
[ Negocia Claves / Firma Desafío ]
[ Establece Canal Seguro ]
[ Exc: Cert Revocado -> Bloquea ]
Validación:
  • Herramientas `testssl.sh`
  • Escaneo `Qualys SSL Labs`
  • Simulacro Phishing MitM (AitM)

🧱 C: Redes (L2-L4)

Técnicas (Requisito Técnico - T):
  • Staging de Reglas (Caso 5)
  • NGFW, DPI y App-ID (Caso 6)
  • Snort IDS pasivo / SPAN (Caso 7)
  • DAI y DHCP Snooping (Caso 9)
  • IPSec, IKEv2, AES-GCM (Caso 10)
Flujo Táctico:
[ Tráfico Entra al Perímetro ]
[ Motor Inspecciona Firmas/Tablas ]
[ Coincidencia / Anomalía ]
[ Ejecuta Acción: Alerta o Bloquea ]
[ Exc: Falso Positivo -> Tuning ]
Validación:
  • Escaneo `Nmap` (Puertos)
  • Inyección `Ettercap` (ARP)
  • Análisis `Wireshark` / Traffic Replay

🌐 D: AppSec (L7)

Técnicas (Requisito Técnico - T):
  • WAF, CSP, Cookies Seguras (Caso 12)
  • Compresión Huffman / Padding (Caso 15)
  • Aislamiento HPACK (Caso 16)
  • Algoritmos Reed-Solomon (Caso 17)
  • LDPC / Belief Propagation (Caso 18)
Flujo Táctico:
[ Recibe Input / Carga Archivo ]
[ Sanitiza / Analiza / Comprime ]
[ Codifica Output / Cifra Fuerte ]
[ Envía / Restaura Respuesta Segura ]
[ Exc: Ataque Oráculo -> Aplica Padding ]
Validación:
  • Pentest DAST (`BurpSuite` / `ZAP`)
  • Test de Canal Lateral (Tamaño payload)
  • Test de Tolerancia a Ruido (SNR)

⚖️ E: GRC y Legal

Técnicas (Requisito Técnico - T):
  • SIEM e Inventario Activos (Caso 19)
  • EIPD y Hardware (Audio OFF) (Caso 21)
  • Hash SHA-2 / Sellado (Caso 22)
  • Logs de Consentimiento (Caso 23)
  • Tag Manager de Bloqueo (Caso 24)
Flujo Táctico:
[ Identifica Riesgo / Muestra Info ]
[ Pide Permiso / Ejecuta PDCA ]
[ Registra Decisión Criptográficamente ]
[ Tratamiento Seguro de Datos ]
[ Exc: Retira Consentimiento -> Cesa ]
Validación:
  • Auditorías Reglamentarias (ISO)
  • Simulacro de Derechos (ARSOPL)
  • Test de Usabilidad Interfaz (UX)
