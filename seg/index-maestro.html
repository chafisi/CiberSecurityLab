Caso Práctico 02

Fuga de Datos por Amenaza Interna

Un empleado descontento descarga bases de datos sensibles de clientes antes de ser despedido, utilizando credenciales válidas.

1. Matriz de Requisitos

Categoría Detalles (Verbos/Métricas/Tecnologías)
Funcionales Detectar descargas masivas, Bloquear cuentas en tiempo real, Alertar a seguridad, Preservar evidencias forenses.
No Funcionales Latencia < 100ms en detección, Tasa de falsos positivos < 1%, Disponibilidad del sistema de monitoreo 99.99%.
Técnicos DLP (Data Loss Prevention), UEBA (User Behavior Analytics), SIEM correlación, Logs de base de datos (SQL Audit), Control DLP en egress.
Legales RGPD (Art. 33 Notificación Brecha), LOPDGDD, Ley de Secretos Empresariales, Normativa ISO 27001 (A.8 Accesos).

Respuestas Tipo: Introducción a la Seguridad

Usa esta plantilla como referencia rápida para cualquier pregunta general sobre fundamentos de seguridad.

Puntos Clave (Palabras Mágicas)

  • Triángulo CIA: Confidencialidad, Integridad, Disponibilidad (siempre mencionarlos).
  • Gestión de Riesgos: Identificar, Evaluar, Tratar, Monitorizar.
  • Respuesta Incidentes: Preparación, Detección, Contención, Erradicación, Recuperación, Lecciones aprendidas.
  • Defensa en Profundidad: Múltiples capas de control, no depender de una sola medida.

Tecnologías Base

  • Cortafuegos (Perimetral/Host-based/NGFW).
  • Antivirus/EDR/XDR básicos.
  • Políticas de Contraseñas y MFA.
  • Backups físicos, lógicos y off-site.
  • SIEM para centralización de logs.

💡 Consejos para Exámenes/Preguntas

  1. Cuando pregunten por "fundamentos", siempre menciona el triángulo CIA primero.
  2. Para preguntas de "gestión", usa la terminología ISO 27001 (políticas, procedimientos, controles).
  3. En casos de "incidentes", sigue la metodología NIST (Preparar, Detectar, Responder, Recuperar).
  4. Siempre justifica tus respuestas citando normativas cuando sea posible (RGPD, ENS, ISO).
