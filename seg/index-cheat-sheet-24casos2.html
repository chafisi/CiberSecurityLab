Modo Estudio: 24 Casos

Haz clic en los bloques borrosos para revelar las respuestas. ¡Pon a prueba tu memoria!

🚨 BLOQUE A: Amenazas Base y Gestión de Incidentes

1. Ransomware (Hospital)

  • RFMonitorizar, aislar red, restaurar backups.
  • RNFRTO < 4h, disponibilidad clínica (99%).
  • TEDR, segmentación L2/L3, backups inmutables.
  • LRGPD (datos salud), Notificación brechas (72h).
🔄 Flujo: Detecta cifrado ➔ EDR aísla ➔ Activa SOC ➔ Restaura Backup
Exp: Continuidad manual
✅ VAL: Simulacro Phishing, Test Restore.

2. Credential Stuffing

  • RFExigir MFA, bloquear cuentas tras N fallos.
  • RNFAuth < 2s, alta escalabilidad portal web.
  • TArgon2/bcrypt, análisis heurístico, WAF.
  • LPSD2 (Banca), RGPD (Integridad).
🔄 Flujo: Usuario loguea ➔ Pide MFA ➔ Valida
Exp: Bloqueo por fuerza bruta
✅ VAL: Hydra/Burp (Fuerza bruta).

20. 0-Day (Log4Shell)

  • RFEvaluar riesgo, inventariar dependencias, parchear.
  • RNFReducir MTTR crítico, gobernanza fluida.
  • TSCA, SAST, monitorización CVSS, IPS.
  • LNIST RMF, DORA (Resiliencia financiera).
🔄 Flujo: Detecta CVE ➔ Identifica activos ➔ Parchea/Aísla
Exp: Mitigación temporal sin parche
✅ VAL: Escáner SCA, Auditoría riesgo residual.
📖 Glosario: RTO (Recovery Time Obj), EDR (Endpoint Det. & Resp.), SOC, MFA, MTTR, SCA (Soft. Comp. Analysis), SAST, CVE, DORA.
🔑 BLOQUE B: Criptografía, Identidad y PKI

3. Transición DES a AES

  • RFUsar algos vigentes, auditar longitudes.
  • RNFRendimiento sin latencia, compatibilidad.
  • TOpenSSL, AES-256, SHA-256.
  • LENS, NIST.
🔄 Flujo: Detecta obsoleto ➔ Migra a AES
Exp: Fallo migración -> Restore
✅ VAL: testssl.sh (Análisis cripto).

4. Certificados Falsos

  • RFValidar estado cert, generar alertas.
  • RNFBaja latencia en verificación (OCSP).
  • TCert Transparency (CT), OCSP, CRL.
  • LeIDAS (Firmas europeas).
🔄 Flujo: Navegador recibe Cert ➔ Valida OCSP/CT
Exp: Revocado -> Bloquea
✅ VAL: Monitorización CT Logs.

13. Certs. Caducados

  • RFSupervisar caducidad, renovar auto.
  • RNFZero-downtime, confiabilidad total.
  • TACME (Let's Encrypt), PKI, Scripts.
  • LRGPD (Cifrado), PCI-DSS.
🔄 Flujo: Detecta expiración ➔ ACME renueva ➔ Despliega
Exp: Fallo -> Alerta manual
✅ VAL: Qualys SSL Labs.

14. MFA FIDO2

  • RFAutenticar FIDO2, registro seguro.
  • RNFUsabilidad alta, multi-dispositivo.
  • TWebAuthn, Hardware Keys, Cripto Asim.
  • LPSD2, RGPD (Biometría).
🔄 Flujo: Loguea ➔ Pide Llave HW ➔ Firma Cripto
Exp: Llave perdida -> Revoca
✅ VAL: Simulacros Phishing MitM.
📖 Glosario: PKI, CRL/OCSP, CT (Cert Transparency), ACME, FIDO2/WebAuthn, eIDAS.
🧱 BLOQUE C: Seguridad Perimetral y Redes (L2-L4)

5. Errores Firewall

  • RFValidar reglas previas, registro cambios.
  • RNFDisponibilidad 99.9%, interfaz segura.
  • TSegmentación, Staging, MFA Admin.
  • LISO 27001 (Control accesos).
🔄 Flujo: Propone regla ➔ Revisa/Prueba ➔ Despliega
Exp: Fallo -> Rollback
✅ VAL: Nmap, Config Drift Detection.

6. Migración NGFW

  • RFInspección DPI, bloquear evasivas (Tor).
  • RNFRendimiento sin latencia DPI, escalable.
  • TApp-ID, descifrado SSL, Integración SIEM.
  • LRGPD (Auditoría de navegación).
🔄 Flujo: Usuario lanza VPN ➔ NGFW DPI detecta ➔ Bloquea
Exp: Falso Positivo
✅ VAL: Test Evasión (Pentest), Tuning SOC.

7. Snort IDS

  • RFDetectar firmas/escaneos, generar alertas.
  • RNFNo perder paquetes (wire-speed).
  • TPuerto SPAN/TAP, Preprocessors, Syslog.
  • LENS (Monitorización contínua).
🔄 Flujo: Snort lee tráfico ➔ Detecta SYN ➔ Alerta SIEM
Exp: Tráfico IT legítimo
✅ VAL: Escaneo con Nmap/Masscan.

8. IPS Financiero

  • RFBloqueo real-time, modo alerta previo.
  • RNFBaja tasa FP, alta estabilidad.
  • THeurística, Despliegue gradual.
  • LPCI-DSS (Protección tarjeta).
🔄 Flujo: Regla alerta ➔ Evalúa FP ➔ Pasa a Bloqueo
Exp: Corta servicio -> Rollback
✅ VAL: Traffic Replay, Tuning iterativo.

9. ARP Spoofing

  • RFVerificar IP-MAC, bloquear suplantación.
  • RNFLatencia nula en Capa 2.
  • TDynamic ARP Inspection (DAI), DHCP Snoop.
  • LRGPD (Prevenir fuga locales).
🔄 Flujo: Switch lee ARP ➔ Valida tabla ➔ Drop inconsistente
✅ VAL: Ataque con Ettercap/arpspoof.

10. IPSec VPN

  • RFTúnel cifrado, renegociar auto claves.
  • RNFEstabilidad de enlace, failover.
  • TIKEv2, AES-GCM, SHA-2, PFS.
  • LRGPD (Transferencia segura).
🔄 Flujo: GW Negocia IKEv2 ➔ Autentica ➔ Túnel ESP
Exp: Falla Auth -> Cierra
✅ VAL: Wireshark (Payload cifrado).
📖 Glosario: DPI (Deep Packet Insp), NGFW, FP (Falso Positivo), IDS/IPS, DAI, IKEv2/ESP, PFS (Perfect Forward Secrecy).
🌐 BLOQUE D: AppSec y Transmisión de Información (L7)

11. Equifax (Parches)

  • RFFlujo formal parcheo, detectar dependencias.
  • RNFMinimizar MTTP, cero regresiones.
  • TSCA, Nessus, Pipeline CI/CD, Staging.
  • LRGPD, PCI-DSS (Cumplimiento parches).
🔄 Flujo: Detecta vulne ➔ Prueba Staging ➔ CI/CD Despliega
Exp: Fallo -> Mitigación WAF
✅ VAL: KPIs de Parcheo (MTTP), Pentest.

12. Robo vía XSS

  • RFSanear inputs, codificar salidas, sesión.
  • RNFRendimiento fluido, Mínimos FP.
  • TCSP, Cookies HttpOnly/Secure, SAST.
  • LPSD2, RGPD (Token = Dato personal).
🔄 Flujo: Mete dato ➔ App Sanitiza/Codifica ➔ Emite HTTP
Exp: Payload -> Drop/Alerta
✅ VAL: BurpSuite (DAST XSS), ZAP.

15. Huffman (JPEG)

  • RFComprimir antes de cifrar, limpiar metadata.
  • RNFCalidad perceptual, baja latencia.
  • TJPEG Huffman, TLS 1.3, Padding uniforme.
  • LRGPD (Privacidad por Diseño).
🔄 Flujo: Sube img ➔ Quita EXIF ➔ Comprime ➔ Cifra TLS
Exp: Oráculo -> Pone Padding
✅ VAL: Análisis Canal Lateral (Tamaño Ciphertext).

16. CRIME

  • RFSeparar secretos de inputs, NO co-comprimir.
  • RNFRendimiento sin TLS compression.
  • THPACK context iso, Padding Random.
  • LISO 27001, RGPD (Evitar fuga cookies).
🔄 Flujo: Req HTTP/2 ➔ Aísla cookies ➔ Cifra sin TLS comp
Exp: HTTP/2 inseguro -> Padding
✅ VAL: Escáner Vulnerabilidad Oráculo.

17. Reed-Solomon

  • RFDetectar/Corregir bits corruptos, loguear BER.
  • RNFFlujo lectura continuo, sin paradas.
  • TAlgoritmo RS (n, k), Buffering óptico.
  • LENS (Retención íntegra a largo plazo).
🔄 Flujo: Lee CD ➔ Detecta daño ➔ RS Reconstruye
Exp: Daño masivo -> Ilegible
✅ VAL: Test físico (Discos dañados).

18. LDPC (Wi-Fi/5G)

  • RFCorregir errores bajo ruido, detectar Jamming.
  • RNFBaja latencia proceso, resiliencia alta SNR.
  • TBelief Propagation, Iteraciones LDPC, WPA3.
  • LNormativas telecomunicaciones (Disp.).
🔄 Flujo: Detecta ruido ➔ Itera LDPC ➔ Reconstruye
Exp: Jamming extremo -> Handover
✅ VAL: Generador Ruido SNR Lab.
📖 Glosario: CI/CD, MTTP, XSS, CSP, HPACK, RS (Reed-Solomon), LDPC (Low-Density Parity-Check), SNR/BER.
⚖️ BLOQUE E: GRC, Legal y Marco Normativo

19. ISO 27001

  • RFEvaluar riesgos, doc políticas.
  • RNFContinuidad.
  • TSIEM, Inventarios.
  • LISO/IEC 27001, RGPD.
🔄 PDCA: PLAN➔DO➔CHECK➔ACT
✅ VAL: Auditoría, KPIs.

21. APDA (CCTV)

  • RFInformar, RAT.
  • RNFMinimización (sin audio).
  • TAudio OFF HW, Cifrado.
  • LLlei 29/2021 Andorra.
🔄 Flujo: EIPD➔Instala sin audio
✅ VAL: Verificar HW Audio.

22. Europol

  • RFPreservar rápido datos.
  • RNFRapidez 24/7.
  • THash SHA-2, Sellos.
  • LConvenio Budapest, MLAT.
🔄 Flujo: Pide➔Preserva➔Firma Hash
✅ VAL: Cadena Custodia.

23. AEPD (RGPD)

  • RFConsentimiento granular.
  • RNFTransparencia, lenguaje.
  • TLogs Inmutables.
  • LRGPD / LOPDGDD España.
🔄 Flujo: Banner➔Acepta➔Log timestamp
✅ VAL: Simulacro ARSOPL.

24. CNIL (Cookies)

  • RF"Rechazar todo" visible.
  • RNFSin manipulación UX/UI.
  • TTag Manager Bloqueo.
  • LePrivacy (Francia).
🔄 Flujo: Banner➔Rechaza➔Bloquea Scripts
✅ VAL: Test Usabilidad (UX).
📖 Glosario: GRC, SGSI, PDCA, APDA/AEPD/CNIL, EIPD, MLAT, ARSOPL (Acceso, Rect., Supr., Opos., Portab., Lim.).
🧠 HACKEANDO EL EXAMEN (1): Patrones de Respuesta por Bloque

No memorices cada caso; memoriza la plantilla de respuesta. Si identificas a qué bloque pertenece la pregunta, el 80% de la respuesta se autocompleta con estas reglas universales:

🧩 Leyes Universales de Clasificación

RF (Funcionales): Qué debe hacer el sistema. Truco: Invierte el ataque. Si el ataque roba -> RF es "Cifrar/Ocultar". Si el ataque paraliza -> RF es "Aislar/Restaurar". Verbos de acción directa.
RNF (No Funcionales): Cómo de bien debe hacerlo. Truco: Siempre usa "Disponibilidad >99%", "Latencia casi nula (wire-speed)", "Usabilidad (no afectar al negocio)", "MTTR/RTO bajos".
T (Técnicos): Con qué tecnología se hace. Truco: Aquí van las siglas técnicas (AES, WAF, EDR, SIEM). Si dudas, pon "Integración con SIEM" que aplica al 90% de los casos.
L (Legales): Qué ley obliga a hacerlo. Truco: "RGPD" es el comodín para datos. "ISO 27001/ENS" para procesos y "PSD2 / DORA" si lees la palabra 'Banca/Finanzas'.

🚨 A: Incidentes

Técnicas Comunes: EDR, Backups, MFA, SIEM.

Flujo Base: Detecta -> Aísla -> Mitiga -> Restaura -> (Exc: Plan Manual).

Validación Común: Simulacros (Phishing/Restore) y Pentest.

🔑 B: Cripto y PKI

Técnicas Comunes: AES-256, FIDO2, ACME, CT/OCSP.

Flujo Base: Inicia conexión -> Valida Identidad/Cert -> Negocia Claves -> (Exc: Revoca/Cierra).

Validación Común: `testssl.sh`, SSL Labs, Revisión de Logs.

🧱 C: Redes (L2-L4)

Técnicas Comunes: NGFW (DPI), Snort, IPSec, DAI.

Flujo Base: Tráfico entra -> Motor analiza firmas -> Coincide -> Bloquea/Alerta -> (Exc: Falso Positivo).

Validación Común: `Nmap` (puertos), `Wireshark` (túneles), Inyección de tráfico.

🌐 D: AppSec (L7)

Técnicas Comunes: WAF, CSP, SAST/DAST, Padding.

Flujo Base: Recibe Input -> Sanitiza/Evalúa -> Codifica Output -> Envía -> (Exc: Drop Payload).

Validación Común: `BurpSuite`/`ZAP` (Pentest web) o Test Canal Lateral.

⚖️ E: GRC y Legal

Técnicas Comunes: Logs inmutables, Hashing (SHA-2), EIPD.

Flujo Base: PDCA (Plan -> Do -> Check -> Act) o Muestra Info -> Pide Permiso -> Registra.

Validación Común: Auditorías internas/externas, Revisión documental.
