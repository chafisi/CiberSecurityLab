Arquitectura y Evolución de Firewalls

Evolución Histórica

1ª Gen (Capa 3 y 4)

Filtro de Paquetes

Analiza cabeceras (IP, Puerto). Estático. Sin memoria de las conexiones. Muy rápido pero fácil de evadir.

2ª Gen (Capa 4)

Stateful Inspection

Guarda el "estado" de la conexión en una tabla de sesiones. Si un paquete no pertenece a una conexión iniciada, se descarta.

3ª Gen (Capa 7)

NGFW & Proxy

Inspección profunda de paquetes (DPI), análisis SSL, filtrado web e integración con IPS. Visibilidad total de las aplicaciones.

Zonas de Seguridad y DMZ

Segmentación física o lógica para contener amenazas basándose en niveles de confianza:

  • LAN (Inside): Máxima confianza. Usuarios corporativos.
  • WAN (Outside): Cero confianza. Internet.
  • DMZ (Desmilitarizada): Zona aislada para servicios públicos (Web, Correo). Si es comprometida, el atacante no puede saltar a la LAN.

Tipos de Despliegue

  • Cortafuegos de Red: Protege un perímetro o segmento completo (Appliance/Router).
  • Cortafuegos de Host: Software en el endpoint (Windows Defender FW). Evita movimientos laterales.
  • Transparente (Bridge): Modo invisible (Capa 2). No altera el enrutamiento IP, filtra tramas Ethernet subrepticiamente.

Configuración: Reglas, ACLs y NAT

Orden de Aplicación de Reglas (ACL)

Los firewalls evalúan las reglas Top-Down (de arriba hacia abajo). La primera regla que hace "Match" se ejecuta y el resto se ignora. La última regla siempre debe ser Implicit Deny (Denegar todo).

NAT (Traducción de Direcciones)

Oculta las IPs internas de la LAN al salir a Internet. El firewall sustituye la IP origen privada por su IP pública.

PAT (Port Address Translation / NAT Overload): Permite que miles de equipos internos compartan una sola IP Pública utilizando diferentes puertos lógicos de salida.

WAF (Web Application Firewall)

Cortafuegos especializado exclusivamente en la Capa 7 para tráfico HTTP/HTTPS.

Bloquea ataques de inyección (SQLi), Cross-Site Scripting (XSS) y anomalías en APIs. Trabaja interpretando el comportamiento de la aplicación web, no solo los puertos.
3

Perspectiva Auditoría: Triada CID

C Confidencialidad

El firewall previene el acceso no autorizado aislando servicios (Segmentación/DMZ). Sin embargo, se enfrenta al reto del tráfico cifrado: si el atacante cifra la exfiltración o el malware (C2), un firewall tradicional está "ciego".

Control de Auditoría: Inspección SSL/TLS profunda (DPI) en NGFW. El firewall actúa como un proxy Man-in-the-Middle autorizado interno para abrir el túnel, leer el contenido, validarlo y volver a cifrarlo hacia el exterior.

I Integridad

La integridad de un cortafuegos depende de la gestión de sus políticas. Una sola regla mal redactada (shadow rule) o un puerto expuesto por error de sintaxis compromete toda la red.

Control de Auditoría: Implementar revisión por pares antes del despliegue, auditorías automatizadas de la tabla de reglas (eliminación de reglas superpuestas) y aplicar siempre el Principio de Mínimo Privilegio.

D Disponibilidad

Los firewalls protegen contra avalanchas de tráfico y conexiones anómalas (SYN Floods). Sin embargo, una mala configuración (Falsos Positivos) o la activación indiscriminada del DPI en NGFW puede agotar la CPU del equipo y colapsar la red legítima.

Control de Auditoría: Escalabilidad y balanceo de carga (HA - Alta Disponibilidad). Monitorizar constantemente el consumo de recursos tras activar análisis heurísticos o de capa 7 para evitar un cuello de botella.
4

Tarjetas de Memorización Flash

Implicit Deny

Regla fundamental al final de cualquier ACL: Si el tráfico no hace match con ninguna regla explícita superior, se bloquea por defecto.

Stateful Firewall

Mantiene una tabla de conexiones activas. Si un paquete de entrada pertenece a una conexión que inició un equipo interno, lo deja pasar dinámicamente.

DMZ

Zona Desmilitarizada. Subred lógica aislada que aloja servicios expuestos a Internet (Web, SMTP) para proteger la red local interna.

NGFW

Next-Gen Firewall. Combina el control clásico con IPS, Inteligencia de Amenazas, DPI (Capa 7) y Control de Aplicaciones.

DPI

Deep Packet Inspection. Análisis del contenido interno del paquete de datos (payload), no solo las cabeceras (IP/Puerto).

WAF

Web Application Firewall. Protege aplicaciones web contra vulnerabilidades como OWASP Top 10 (Inyecciones SQL, XSS).

C5

Caso 5: Mala config (Perimetral)

Enunciado del Caso

Una mala configuración en un firewall perimetral dejó expuesto a Internet un puerto de BBDD. Falta de segmentación permitió acceso por fuerza bruta provocando exfiltración de datos. La auditoría reveló que la gestión era manual, no había revisión por pares, ni procedimientos formales, ni detección de puertos expuestos.

1. Requisitos para evitar errores de configuración

Funcionales / No Funcionales:
  • Prevenir exposición accidental de servicios hacia redes no confiables.
  • Aislar activos críticos (segmentación).
  • Garantizar disponibilidad ante incidentes.
Técnicos / Organizativos:
  • Detección automatizada de puertos expuestos.
  • Diseño bajo mínimo privilegio.
  • Procedimientos formales y revisión por pares (evitar modo manual).

2. Caso de Uso: Gestión segura de reglas

Actor: Administrador de red y Auditor (revisión conjunta).

Flujo principal:
  1. El admin diseña una regla con "mínimo privilegio".
  2. Se envía a herramienta de gestión (no manual).
  3. Un auditor realiza "revisión por pares".
  4. Se aplican "procedimientos formales de validación" en preproducción.
  5. Se despliega y se monitoriza.
Excepciones:
  • La revisión detecta un puerto genérico expuesto → Se bloquea el pase a producción y se exige segmentación.

3. Métodos de validación

  • Escaneos de vulnerabilidades: Proceso automatizado de detección empírica desde el exterior.
  • Simulación Fuerza Bruta: Verificar bloqueos temporales e IPS.
  • Auditorías periódicas: Depurar reglas obsoletas (shadow rules).
  • Logs de ticketing: Comprobar evidencias de revisión por pares.

💡 Correlación Ignacio (Ingeniería Inversa)

Pista (Enunciado) Solución
"...no existían procedimientos formales para validar reglas..." Gobernanza estricta sobre cambios (Req. Organizativo).
"...no había revisión por pares..." Participación obligatoria en el Flujo Principal (Caso Uso).
"...proceso automatizado detección de puertos expuestos." Validación mediante Escaneos automatizados desde el exterior.
C6

Caso 6: Migración a NGFW

Enunciado del Caso

Migración a NGFW para detectar apps evasivas (Tor, proxies anónimos) que saltan restricciones corporativas. Se integró inspección profunda de paquetes (DPI), control de aplicaciones y análisis de tráfico cifrado para ver patrones encubiertos. Requirió ajustes de rendimiento, optimización de políticas y formación del personal (alertas/capacidades). Requiere gestión del cambio.

1. Requisitos para la transición a NGFW

Funcionales / No Funcionales:
  • Bloquear tráfico encubierto (evasivas).
  • Aplicar control de apps mediante DPI (Capa 7).
  • Asegurar rendimiento de red (sin cuellos botella).
Técnicos / Organizativos:
  • Infraestructura PKI para análisis de tráfico cifrado.
  • Gestión del cambio organizativo.
  • Capacitación continua del SOC.

2. Caso de Uso: Detección apps no autorizadas

Actor: Motor NGFW (Inspección Capa 7).

Flujo principal:
  1. Empleado ejecuta herramienta evasiva (proxy anónimo).
  2. Firewall aplica "análisis de tráfico cifrado" abriendo el túnel.
  3. Se realiza "DPI" sobre el payload.
  4. El sistema identifica "patrones encubiertos" (firmas Tor).
  5. Bloquea "conexiones no autorizadas" y alerta al SOC.
Excepciones:
  • Tráfico pertenece a banca/salud (exento de inspección) → Omite descifrado y aplica filtro estándar SNI.

3. Métodos de validación de eficacia

  • Inyección de tráfico simulado: Usar Tor/proxies controlados para confirmar el bloqueo en Capa 7.
  • Monitorización de rendimiento: Comprobar uso de CPU/RAM en horas pico para validar la "optimización de políticas".
  • Red Teaming: Validar "canales cifrados" y descifrado SSL sin errores de certificado.
  • Ejercicios de Incident Response: Validar que el SOC ha asimilado la "formación del personal".

Glosario de Términos (U4)

ACL

Access Control List. Lista secuencial de reglas (Permit/Deny) aplicadas top-down.

DMZ

Demilitarized Zone. Segmento intermedio que aísla los servicios expuestos a Internet para proteger la red interna.

DPI

Deep Packet Inspection. Análisis profundo del cuerpo (payload) del paquete, buscando firmas o malware, no solo cabeceras.

Implicit Deny

Regla final invisible en un firewall: "Lo que no está explícitamente permitido, está prohibido".

NAT / PAT

Network/Port Address Translation. Traduce direcciones IP privadas a públicas. PAT usa puertos para compartir una sola IP pública.

NGFW

Next-Generation Firewall. Integra inspección de estado con inteligencia de aplicaciones (Capa 7), IDS/IPS y descifrado SSL.

Shadow Rules

Reglas superpuestas. Ocurre cuando una regla más amplia colocada arriba impide que una regla más específica abajo llegue a evaluarse.

Stateful Firewall

Cortafuegos con memoria. Rastrea el estado de las conexiones activas, permitiendo dinámicamente el tráfico de respuesta.

WAF

Web Application Firewall. Defiende aplicaciones web de inyecciones, XSS y abusos HTTP, actuando como escudo inverso.
