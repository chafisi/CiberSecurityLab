1. Introducción a la Minería de Datos (KDD)

A diferencia del Big Data (que trata sobre el volumen y almacenamiento), la Minería de Datos es el proceso de analizar grandes conjuntos de datos para descubrir patrones ocultos, extraer información valiosa y generar conocimiento. Es el núcleo de la metodología KDD (Knowledge Discovery in Databases).

Objetivos Principales y Tipos de Tareas

1. Descripción

Entender qué ha pasado resumiendo la información y encontrando patrones interpretables por humanos.

2. Predicción

Anticipar eventos futuros o valores desconocidos basándose en el análisis de datos históricos.

Técnicas Clave de Minería:

  • Clasificación (Supervisada) Asigna elementos a categorías predefinidas.
  • Clustering (No supervisada) Agrupa elementos similares sin etiquetas previas.
  • Regresión (Supervisada) Estima o predice un valor numérico continuo.
  • Asociación Descubre reglas "si-entonces" (ej. "Si compra pan, comprará leche").

2. Preprocesamiento y Limpieza de Datos

El Principio GIGO: "Garbage In, Garbage Out"

Ningún algoritmo avanzado puede compensar datos de mala calidad. El preprocesamiento consume habitualmente el 70-80% del tiempo de un proyecto de minería.

🧹 Limpieza e Integración

  • Integración: Fusionar datos de múltiples fuentes heterogéneas (ERP, CRM, LMS) en un almacén coherente.
  • Valores Faltantes: Decidir entre eliminar registros incompletos o aplicar imputación (sustituir por media, mediana o algoritmos).
  • Ruido y Outliers: Identificar y corregir valores atípicos que pueden sesgar el modelo (ej. una edad de "999" años).

⚙️ Transformación y Reducción

  • Normalización: Escalar variables para que estén en el mismo rango (ej. 0 a 1) para evitar que variables grandes dominen a las pequeñas.
  • One-Hot Encoding: Convertir variables categóricas (ej. Color: Rojo/Azul) en columnas binarias (1 o 0) entendibles por la máquina.
  • Reducción de Dimensionalidad: Usar técnicas como PCA para reducir la cantidad de variables manteniendo la información esencial.

3. Técnicas Estadísticas Avanzadas

Técnicas No Supervisadas

Agrupan datos basándose exclusivamente en su similitud matemática, sin disponer de etiquetas (respuestas) previas.

Algoritmo K-Means:
  1. Elegir K (número de clusters) usando Método del Codo o Silueta.
  2. Posicionar K centroides aleatorios.
  3. Asignar cada dato al centroide más cercano.
  4. Recalcular el centroide basándose en la media del grupo. Repetir hasta converger.
Agrupamiento Jerárquico:

Crea una jerarquía de clústeres representable mediante un Dendrograma.

Técnicas Supervisadas

Aprende de un conjunto de datos histórico que ya contiene las respuestas correctas (etiquetas) para predecir futuros valores.

Regresión Lineal:

Simple ($y = mx + b$) o Múltiple. Busca la línea recta que minimiza la distancia con todos los puntos de datos.

Validación y Riesgos:
  • RMSE: Raíz del Error Cuadrático Medio. (Menor es mejor).
  • R²: Coeficiente de determinación (0 a 1).
  • Overfitting: El modelo memoriza los datos de entrenamiento y falla con datos nuevos.

Caso Práctico 15: Segmentación en E-commerce (Clustering)

Solución Oficial

Enunciado: Una empresa de comercio electrónico quiere entender mejor el comportamiento de compra para mejorar sus estrategias de marketing. Describe cómo usar técnicas de clustering para segmentar a sus clientes.

Flujo de Solución:

  • Preprocesamiento: Recolectar historial de compras, frecuencia, valor promedio. Limpiar y normalizar datos para asegurar la misma escala.
  • Algoritmo (K-Means): Paso 1: Determinar clusters (K) usando método del codo. Paso 2: Asignar cliente al centroide cercano. Paso 3: Recalcular centroides hasta que no cambien.
  • Interpretación: Analizar perfiles (ej. alta frecuencia/bajo valor vs alta valor/baja frecuencia).
  • Aplicación de Negocio: Personalizar campañas de marketing, gestión de inventario predictivo y retención de clientes clave.

Caso Práctico 16: Predicción de Estancia Hospitalaria (Regresión)

Solución Oficial

Enunciado: Un hospital quiere predecir la duración de estancia de los pacientes para optimizar la gestión de camas y recursos. Describe cómo usar técnicas de regresión para este fin.

Flujo de Solución:

  • Recolección: Datos históricos (edad, diagnóstico, pruebas) y duración de estancia real (Variable Objetivo).
  • Preprocesamiento: Manejar faltantes y aplicar One-Hot Encoding a diagnósticos categóricos.
  • Modelado (Regresión Lineal Múltiple): Dividir en entrenamiento y prueba. Usar mínimos cuadrados para ajustar coeficientes predictores.
  • Validación: Evaluar con métricas como RMSE o $R^2$ para verificar precisión.
  • Aplicación: Usar la fórmula generada para predecir días en nuevos ingresos, optimizando la planificación hospitalaria.

5. Fichas Rápidas (Conceptos Core)

🎯

K-Means

Algoritmo

Agrupamiento no supervisado basado en iterar la distancia hacia centroides dinámicos.

📉

Método del Codo

Evaluación

Técnica gráfica para encontrar el número óptimo de clusters buscando la inflexión en la varianza.

📈

Regresión Lineal

Modelo Predictivo

Modela matemáticamente la relación lineal entre variables independientes y una dependiente.

📏

RMSE

Métrica de Error

Raíz del Error Cuadrático Medio. Penaliza grandes desviaciones; a menor valor, mejor modelo.

🧠

Overfitting

Sobreajuste

Cuando el modelo memoriza el ruido del entrenamiento y falla al generalizar datos nuevos.

🔢

One-Hot Encoding

Transformación

Convierte variables categóricas (texto) en múltiples columnas binarias de 0 y 1.

⚖️

Normalización

Preprocesamiento

Escalar datos numéricos a un rango estándar (ej. 0 a 1) evitando sesgos de magnitud.

📍

Cluster Centroid

Centro Geométrico

El punto central representativo de un clúster calculado como la media de sus integrantes.

6. Glosario Operativo

Feature Engineering

Creación matemática de nuevas variables a partir de datos existentes para darle más "señal" al modelo.

Label (Etiqueta)

En aprendizaje supervisado, es la variable objetivo que queremos que el modelo aprenda a predecir.

Training Set / Test Set

División estricta de datos; una porción para enseñar al modelo y otra oculta para examinar su precisión.

Distance Metric

Fórmula matemática (como Euclidiana o Manhattan) usada para medir qué tan similares son dos registros.

Confusion Matrix

Tabla evaluativa de clasificadores que desglosa Verdaderos/Falsos Positivos y Negativos.

Imputación y Outliers

Imputar es rellenar huecos vacíos con estimaciones. Outliers son datos anómalos que distorsionan el aprendizaje.

El ciclo de vida estándar en Minería de Datos (basado en metodologías como CRISP-DM). Haz clic para expandir.

🏢

1. Entendimiento del Negocio

Definición de Objetivos
▼

⬇
📊

2. Comprensión de Datos

Exploración (EDA)
▼
Recolección inicial y análisis exploratorio. Verificación de la calidad, distribución de variables y detección visual de anomalías.
⬇
🧹

3. Preparación de Datos

Limpieza y Transformación (70% del tiempo)
▼
Imputación de nulos, eliminación de outliers, normalización y One-Hot Encoding. Aquí se define la calidad del modelo (GIGO).
⬇
⚙️

4. Modelado

Entrenamiento de Algoritmos
▼
Selección de técnica (Regresión, K-Means, etc.). Se entrena el algoritmo ajustando sus hiperparámetros.
⬇
🎯

5. Evaluación y Despliegue

Validación y Producción
▼
Cálculo de métricas (RMSE, R2). Si supera las expectativas del negocio, el modelo se integra en sistemas de producción y se monitoriza su desviación en el tiempo (Data Drift).
