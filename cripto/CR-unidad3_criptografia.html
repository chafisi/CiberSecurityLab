APARTADO 1

1. Fundamentos Matemáticos y Evolución del Algoritmo

Propiedades de las Funciones Hash

Las funciones Hash son algoritmos matemáticos que transforman un bloque de datos de longitud variable en una cadena de longitud fija (resumen o *digest*). Sus pilares son:

  • Determinismo: La misma entrada siempre producirá exactamente el mismo hash de salida.
  • Unidireccionalidad: Es computacionalmente inviable deducir el mensaje original a partir del hash generado.
  • Efecto Avalancha: Un cambio mínimo en la entrada (incluso un solo bit) altera drásticamente el hash resultante.
  • Resistencia a Colisiones: Es altamente improbable encontrar dos mensajes diferentes que generen el mismo hash.
Simetría Avanzada: El Estándar AES

El *Advanced Encryption Standard* (AES) sustituyó a DES, solventando sus vulnerabilidades de fuerza bruta y tamaño de clave.

  • Estructura Operativa: No utiliza redes de Feistel, sino una red de Sustitución-Permutación (Rijndael).
  • Bloques: Opera dividiendo los datos en matrices de estado de tamaño fijo de 128 bits.
  • Claves y Rondas: Soporta longitudes de clave de 128, 192 o 256 bits. A mayor clave, mayor número de rondas de transformación matemática (10, 12 y 14 rondas respectivamente), garantizando la confidencialidad.
APARTADO 2

2. Implementación Algorítmica y Despliegue Técnico

Flujo Lógico: Integridad vs Confidencialidad

Para Integridad (SHA-256): El texto original es sometido a operaciones lógicas complejas (AND, OR, XOR, ROT, SHR) y sumas modulares a lo largo de 64 rondas para emitir un resumen irreversible de 256 bits.

Para Confidencialidad (AES): El texto plano se divide en bloques de 128 bits. En cada ronda de cifrado se aplican cuatro pasos: SubBytes (sustitución no lineal), ShiftRows (desplazamiento de filas), MixColumns (mezcla lineal de columnas) y AddRoundKey (XOR con subclave derivada).

Comandos de Laboratorio (OpenSSL)

# 1. Generar un Hash SHA-256 para verificar la integridad de un archivo de datos
openssl dgst -sha256 dataset_edushield.csv

# 2. Cifrar un archivo usando AES de 256 bits (Confidencialidad)
# Se solicitará una contraseña para derivar la clave simétrica.
openssl enc -aes-256-cbc -salt -in reporte_patito.pdf -out reporte_cifrado.enc

# 3. Descifrar el archivo AES usando la contraseña compartida
openssl enc -aes-256-cbc -d -in reporte_cifrado.enc -out reporte_recuperado.pdf

3. Fichas de Repaso (Tarjetas de Memorización)

Efecto Avalancha
Definición: Propiedad criptográfica donde un pequeño cambio en la entrada (un bit) altera al menos la mitad de los bits de salida.

Ejemplo: Hash("Hola") vs Hash("hola") son completamente distintos.
Colisión Hash
Definición: Sucede cuando dos entradas de datos completamente diferentes generan la misma huella digital (hash).

Ejemplo: La principal vulnerabilidad que hizo obsoleto a MD5.
Matriz de Estado (AES)
Definición: Estructura interna de 4x4 bytes (128 bits) donde AES realiza sus operaciones de sustitución y permutación.

Ejemplo: ShiftRows mueve las filas de esta matriz para aplicar difusión.
Unidireccionalidad
Definición: Es fácil calcular la salida desde una entrada, pero computacionalmente imposible calcular la entrada desde la salida.

Ejemplo: La base del almacenamiento seguro de contraseñas.

4. Casos Prácticos: Soluciones Oficiales Integradas

Caso Práctico 5: Integridad con SHA-256

Contexto: Alice quiere asegurar que un mensaje enviado a Bob no ha sido alterado. Decide usar SHA-256. Describe el proceso.

Solución Oficial:

  • Generación del Hash: Antes de enviar el mensaje, Alice calcula el hash del mensaje usando SHA-256 (produce 256 bits). Es resistente a colisiones.
  • Transmisión: Alice envía el mensaje junto con su hash SHA-256 a Bob.
  • Verificación por Bob: Bob recibe ambos. Genera el hash del mensaje recibido usando SHA-256 y lo compara con el enviado por Alice.
  • Seguridad: Si coinciden, el mensaje no ha sido alterado. Se basa en la resistencia a colisiones de SHA-256 y la imposibilidad de revertirlo.
Caso Práctico 6: Confidencialidad con AES

Contexto: Bob necesita asegurar que solo Alice pueda leer un archivo críptico. Decide usar AES. Describe el cifrado y descifrado.

Solución Oficial:

  • Generación de la Clave: Bob y Alice comparten una clave secreta de 128, 192 o 256 bits. La seguridad depende de mantenerla en secreto.
  • Cifrado (Bob): AES opera en bloques de 128 bits. El archivo se divide y se aplican múltiples rondas de sustituciones y permutaciones combinadas con la clave. Bob transmite el archivo.
  • Descifrado (Alice): Alice usa la misma clave secreta. El proceso es el inverso del cifrado (operaciones en orden inverso).
  • Confidencialidad: Solo quien tiene la clave correcta puede descifrar, asegurando el acceso exclusivo a Alice.

5. Glosario Técnico

SHA-256 (Secure Hash Algorithm)
Algoritmo de la familia SHA-2 que genera un resumen criptográfico de 256 bits. Es el estándar actual en la industria para firmas digitales, certificados TLS y tecnología Blockchain (Bitcoin).
AES (Advanced Encryption Standard)
El estándar global para el cifrado de datos simétricos. Fue adoptado por el NIST en 2001 (Rijndael). Utiliza longitud de bloques fija y es altamente eficiente tanto en software como en hardware.
Determinismo Criptográfico
La garantía algorítmica de que, en las funciones hash, si se ingresa exactamente el mismo conjunto de datos bajo las mismas condiciones, se generará indefectiblemente el mismo hash de salida.
S-Box (Caja de Sustitución)
Componente fundamental de algoritmos de clave simétrica como AES que introduce no linealidad en el cifrado, haciendo que sea resistente al criptoanálisis lineal y diferencial.
