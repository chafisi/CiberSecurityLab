1. Funciones Hash: Fundamentos de Integridad

🔍 Propiedades Esenciales

  • Determinismo: La misma entrada de datos siempre producirá exactamente el mismo hash.
  • Unidireccionalidad: Es computacionalmente inviable reconstruir el mensaje original a partir de su hash.
  • Efecto Avalancha: Modificar un solo bit en la entrada altera drásticamente el resultado final.

🛡️ Casos de Uso Reales

  • Integridad de Archivos: Validar descargas comprobando el checksum SHA-256.
  • Contraseñas: Almacenamiento de credenciales seguras (Hash + Salt) en bases de datos.
  • Blockchain: Minería de bloques y creación de cadenas inmutables.

2. Encriptación Simétrica Avanzada (AES)

⏳

De DES a AES

El antiguo estándar DES operaba con bloques de 64 bits y claves de 56 bits (vulnerables a fuerza bruta). AES fue seleccionado por el NIST como estándar superior, utilizando un algoritmo de sustitución-permutación llamado Rijndael.

🧱

Arquitectura de Bloques y Rondas

AES opera siempre dividiendo los datos en matrices de estado de 128 bits (4x4 bytes). Soporta claves de 128, 192 o 256 bits. A mayor tamaño de clave, mayor número de rondas de cifrado (10, 12 y 14 rondas respectivamente).

4. Fichas Rápidas (Memorización)

🏔️

Efecto Avalancha

Propiedad vital del Hash y AES

Definición

Si se cambia un solo bit en la entrada (ej. una "A" por una "a"), el hash resultante o bloque cifrado cambiará de forma irreconocible (más del 50%).

💥

Colisión

Vulnerabilidad en Funciones Hash

Definición

Ocurre cuando el algoritmo produce exactamente el mismo hash para dos mensajes de origen completamente distintos. (El motivo por el que MD5 ya no se usa).

📦

Matriz de Estado

Estructura interna de AES

Definición

En AES, los datos no se cifran en cadena plana, sino que se mapean en una cuadrícula de 4x4 bytes (128 bits) donde se aplican desplazamientos de filas y mezcla de columnas.

🔀

S-Box (Sustitución)

Fase no lineal de cifrado

SubBytes

Es una tabla de búsqueda (LUT) matemática que reemplaza cada byte del estado por otro. Aporta la "confusión" necesaria para frustrar el criptoanálisis.

Caso 5: Integridad con SHA-256

Solución Plataforma

Enunciado: Alice quiere asegurar que un mensaje enviado a Bob no ha sido alterado durante la transmisión. Decide usar una función hash. Describe cómo usar SHA-256 para este propósito.

Secuencia Operativa:

  • Generación del Hash: Antes de enviar el mensaje, Alice calcula el hash del mensaje usando SHA-256 (produce 256 bits). Es prácticamente imposible encontrar dos mensajes diferentes con el mismo hash (resistencia a colisiones).
  • Transmisión: Alice envía el mensaje junto con su hash SHA-256 a Bob.
  • Verificación (Bob): Al recibir el mensaje, Bob genera el hash del mensaje recibido usando SHA-256 de manera independiente.
  • Garantía de Seguridad: Si el hash calculado coincide con el enviado por Alice, Bob puede estar seguro de que el mensaje no ha sido alterado. Esto se basa en la resistencia a colisiones de SHA-256 y su unidireccionalidad.

ESQUEMA 5: SHA-256 (INTEGRIDAD)

  • 1️⃣ HASHING ⚙️ Alice calcula Hash = SHA-256(M).
  • 2️⃣ TRANSMISIÓN 📤 Envía [Mensaje Original] + [Hash_A].
  • 3️⃣ VERIFICACIÓN 🕵️‍♂️ Bob calcula Hash_B con el mensaje recibido.
  • 4️⃣ VALIDACIÓN ✅ ¿Hash_A == Hash_B? (Sí=Íntegro).

Caso 6: Confidencialidad con AES

Solución Plataforma

Enunciado: Bob necesita asegurar que solo Alice pueda leer un archivo críptico que él le enviará. Decide usar AES. Describe cómo Bob cifraría el archivo y cómo Alice lo descifraría.

Secuencia Operativa:

  • Acuerdo de Clave: Bob y Alice comparten de antemano una clave secreta (AES soporta 128, 192 o 256 bits). La seguridad recae completamente en proteger esta clave.
  • Cifrado (Bob): Bob usa AES, que opera dividiendo el archivo en bloques de 128 bits. A cada bloque se le aplican múltiples rondas (sustituciones, permutaciones y combinaciones con la clave). Posteriormente transmite el archivo cifrado.
  • Descifrado (Alice): Alice usa la misma clave secreta. El proceso aplica las mismas operaciones de cifrado pero en orden inverso para recuperar el original.
  • Confidencialidad: Solo alguien con la clave simétrica correcta puede descifrar el bloque de datos.

ESQUEMA 6: AES (CONFIDENCIALIDAD)

  • 1️⃣ CLAVE 🗝️ Acuerdo previo de clave (128/192/256 bits).
  • 2️⃣ CIFRADO 🔒 División en bloques (128 bits) y rondas S-Box + Permutación.
  • 3️⃣ TRANSMISIÓN 📡 Envío de criptograma (ruido inútil).
  • 4️⃣ DESCIFRADO 🔓 Alice aplica clave + proceso AES inverso.

🤐 Sólo el portador de la clave lee el archivo.

6. Glosario Técnico

Integridad de Datos

Garantía de que la información no ha sido modificada, alterada o destruida de forma no autorizada durante su almacenamiento o transmisión (Objetivo del Hash).

Confidencialidad

Garantía de que la información solo es accesible por personas, entidades o procesos autorizados (Objetivo de la encriptación AES).

Cifrado de Bloque (Block Cipher)

Método de cifrado simétrico que opera en grupos de bits de longitud fija llamados bloques, a diferencia del cifrado de flujo (Stream Cipher).

Rondas (Rounds en AES)

Iteraciones del proceso matemático de confusión y difusión que AES aplica sobre la matriz de estado. Las rondas se repiten 10, 12 o 14 veces según la clave.
