El Esqueleto Común (Proceso KDD)

Empieza siempre tus respuestas estructurando la solución en estos 5 apartados. Esto demuestra que entiendes el ciclo de vida del dato completo.

📥

1. Recolección

Definir qué variables (Inputs) tenemos. Si es supervisado, definir también el objetivo (Output).

🧹

2. Preprocesamiento

Limpiar nulos, categorizar y Normalizar (vital para que el modelo no se sesgue).

¡CRÍTICO!
⚙️

3. Modelo

Aplicar el algoritmo (Regresión, Árbol o K-Means). Dividir en Train/Test si hay etiquetas.

⚖️

4. Evaluación

Medir si el modelo funciona bien usando la métrica adecuada (RMSE, Matrices, etc.).

🚀

5. Aplicación

¿Para qué sirve al negocio? Toma de decisiones, automatización de alertas, optimización.

Rellenando el paso "3 y 4" según la técnica

Supervisado

Regresión Lineal

1. RECOLECCIÓN

  • Datos históricos.
  • Variables independientes y dependiente (objetivo a adivinar).
  • Limpiar nulos | Texto a num (One-hot encoding).

2. MOTOR / ENTRENAMIENTO

  • Divide siempre: Entrenamiento (80%) | Prueba (20%).
  • Ajuste de pesos/coeficientes por mínimos cuadrados.

3. EVALUACIÓN

  • RMSE (Raíz del Error Cuadrático Medio): Mide la distancia de error promedio entre la predicción y el valor real (se busca el mínimo).
  • R²: Porcentaje de acierto o varianza explicada.

4. APLICACIÓN

  • Predecir un número exacto para anticipar decisiones (Ej: nota, días, precio).
Supervisado
🌳

Árbol de Decisión

1. RECOLECCIÓN

  • Históricos con categorías dadas.
  • Clasifica en 2 categorías (Ej: Cat 1 "Activo" | Cat 0 "Riesgo").

2. MOTOR / ENTRENAMIENTO

  • Divide siempre: Entrenamiento (80%) | Prueba (20%).
  • Crea ramas dividiendo el dataset con preguntas lógicas (Gini).

3. EVALUACIÓN

  • Precisión y Recall.
  • Matriz de Confusión: Para visualizar verdaderos aciertos vs falsos positivos/negativos.

4. INTERPRETACIÓN / APLICACIÓN

  • Muestra reglas claras legibles por humanos (Si tiempo < 2h y tareas < 50% -> Riesgo).
  • Envío de alertas tempranas y planes personalizados.

Clusters (K-Means)

1. PREPROCESAMIENTO

  • Datos puros (sin etiquetas ni objetivo previo).
  • Limpiar | Normalizar (crítico para las distancias).

2. MOTOR / ALGORITMO

  • No hay división de entrenamiento/prueba.
  • Paso 1: Elegir 'K' (nº de grupos) mediante el Método del Codo.
  • Paso 2: Coloca centroides aleatorios.
  • Paso 3: Agrupa puntos por cercanía iterando hasta converger.

3. EVALUACIÓN / INTERPRETACIÓN

  • Analizar los centroides finales y etiquetar grupos por patrones (humanizar el dato).
  • Ej: Segmento "Caza ofertas" o "Ocasionales".

4. BENEFICIOS

  • Personalización de Marketing, descubrimiento de perfiles ocultos y satisfacción del cliente.
📈

Regresión Lineal

1. RECOLECCIÓN Y PREPARACIÓN

  • Datos históricos.
  • Independientes y dependiente (objetivo).
  • Limpiar nulos | Texto a num (One-hot encoding).

2. ENTRENAMIENTO MODELO

  • Regresión lineal (Objetivo | Resto factores).
  • Entrenamiento: Dataset 80% / 20% para validar.

3. EVALUACIÓN

  • Precisión modelo RMSE o R².

4. APLICACIÓN

  • Predicción | Ayuda a la toma de decisiones.
🎯

Clusters (K-Means)

1. PREPROCESAMIENTO

  • Recolectar datos.
  • Limpiar | Normalizar (importante).

2. ENTRENAMIENTO MODELO

  • Algoritmo clustering K-means.
  • Agrupar por características.
  • Paso 1: K clusters (método codo) | Paso 2: Asignar centroides cercanos | Paso 3: Iterar.

3. EVALUACIÓN / INTERPRETACIÓN

  • Etiquetar grupos por patrones.
  • Ej: "Caza ofertas" o "Ocasionales".

4. BENEFICIOS

  • Uso de datos Marketing | Análisis mercado | Satisfacción clientes.
🌳

Árbol de Decisión

1. DATOS

  • Históricos dados.

2. MODELO

  • Clasifica en 2 categorías:
  • Cat 1 (1): "Activo" | Cat 2 (0): "Riesgo" (según enunciado).

3. ENTRENAMIENTO

  • Divide en Entrenamiento | Prueba.
  • Evalúa: Precisión | Recall | Matriz de confusión.

4. INTERPRETACIÓN

  • Muestra reglas claras.
  • Ej: Si tiempo conex < 2h/sem y % tareas < 50% -> Riesgo.

5. APLICACIÓN Y BENEFICIOS

  • Envío de alertas profesores | Diseño planes personalizados.
  • Detección temprana.

Regresión Lineal / Múltiple

Supervisado

Objetivo: Predecir un número continuo (ej. Nota final, días de estancia, precio, tiempo de entrega).

  • Paso 1 y 2 (Datos): Tenemos histórico de datos. Transformamos texto a números (One-Hot).
  • Paso 3 (Motor): Ajusta una línea matemática a través de los datos mediante Mínimos Cuadrados para predecir el valor Y.
  • Paso 4 (Evaluación): Se mide el error. Usamos RMSE (Raíz del Error Cuadrático Medio) y R².
🎯

Clustering (K-Means)

No Supervisado

Objetivo: Descubrir perfiles o segmentos ocultos en una base de datos sin etiquetas previas.

  • Paso 1 y 2 (Datos): Datos puros sin etiquetas. Vital normalizar para que el cálculo de distancias sea justo.
  • Paso 3 (Motor): Se elige 'K' (nº de grupos). El algoritmo coloca centroides y agrupa los puntos por cercanía iterativamente.
  • Paso 4 (Evaluación): Método del Codo para elegir K. Validación con el Coeficiente de Silueta (cohesión interna de los grupos).
🌳

Árboles de Decisión

Supervisado

Objetivo: Clasificar en categorías (ej. Abandona/No Abandona, Spam/No Spam, Riesgo Alto/Bajo).

  • Paso 1 y 2 (Datos): Tenemos datos históricos con sus etiquetas (categorías ya conocidas).
  • Paso 3 (Motor): Divide los datos iterativamente basándose en preguntas lógicas usando la Impureza de Gini o Entropía ("Si X < 5, entonces...").
  • Paso 4 (Evaluación): Medimos aciertos y fallos usando la Matriz de Confusión, Precisión y Recall.
