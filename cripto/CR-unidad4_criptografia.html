1. Fundamentos: Claves Públicas y Privadas

🗝️ El Paradigma Asimétrico

A diferencia de la criptografía simétrica (donde se comparte una sola clave), la criptografía asimétrica utiliza un par de claves matemáticamente vinculadas.

  • Clave Pública: Se distribuye libremente. Se usa para cifrar mensajes hacia el propietario o verificar sus firmas.
  • Clave Privada: Debe permanecer en secreto absoluto. Se usa para descifrar mensajes recibidos o firmar digitalmente.

🏆 Ventajas Clave

  • Cero intercambio previo: No es necesario arriesgarse a transmitir una clave secreta por un canal inseguro.
  • Autenticación (No repudio): Si un mensaje se cifra con la clave privada de Alice, y todos pueden leerlo con su clave pública, es matemáticamente indudable que Alice lo envió.
  • Mitigación de MITM: Protege contra ataques de Intermediario al establecer canales seguros (ej. TLS/SSL).

2. Arquitectura de las Firmas Digitales RSA

¿Cómo funciona la Firma Digital? (Autenticidad)

La firma digital invierte el proceso de confidencialidad. Aquí, el emisor utiliza su propia clave Privada para "sellar" matemáticamente el documento.

  1. El emisor calcula el Hash (SHA-256) del mensaje original.
  2. El emisor cifra ese Hash usando su Clave Privada (d). El resultado es la Firma Digital (S).
  3. Se transmite el Mensaje original junto con la Firma Digital (S).
  4. El receptor calcula su propio Hash del mensaje. Además, descifra la Firma Digital (S) usando la Clave Pública (e) del emisor.
  5. Si ambos Hashes coinciden, la firma es válida: el mensaje es auténtico y no ha sido alterado.

Crear Firma (Emisor)

S = H(M)d mod n

Verificar Firma (Receptor)

H(M)' = Se mod n

3. Implementación Técnica (OpenSSL)

rsa-firmas.sh

# 1. Generar par de claves RSA (Alice)

openssl genrsa -out alice_privada.pem 2048
openssl rsa -in alice_privada.pem -pubout -out alice_publica.pem

# 2. Firmar digitalmente un documento (Genera el Hash SHA256 y lo cifra con la clave Privada)

openssl dgst -sha256 -sign alice_privada.pem -out firma.sig contrato.pdf

# 3. Verificar la firma digital (El receptor usa la clave Pública de Alice y el documento original)

openssl dgst -sha256 -verify alice_publica.pem -signature firma.sig contrato.pdf

# Output esperado: "Verified OK"

Caso Práctico 7: Mensaje Secreto (Confidencialidad RSA)

Solución Oficial

Enunciado: Alice quiere enviar un mensaje secreto a Bob utilizando RSA. Describe el proceso de generación de claves por parte de Bob y cómo Alice cifra el mensaje para que solo Bob pueda leerlo.

Respuesta de la Plataforma:

  • Generación de Claves por Bob:
    • Bob elige dos números primos grandes p y q.
    • Calcula n = p * q y φ(n) = (p - 1)(q - 1).
    • Selecciona un número público e tal que 1 < e < φ(n) y gcd(e, φ(n)) = 1.
    • Calcula la clave privada d tal que d * e ≡ 1 mod φ(n), usando el algoritmo de Euclides extendido.
    • La clave pública es (e, n) y la clave privada es (d, n).
  • Cifrado por Alice: Alice obtiene la clave pública de Bob (e, n). Convierte su mensaje en un número M (usando algún esquema de codificación). Cifra el mensaje con la fórmula C = Me mod n, donde C es el mensaje cifrado.
  • Transmisión: Alice envía el mensaje cifrado C a Bob.
  • Descifrado por Bob: Bob usa su clave privada (d, n) para descifrar el mensaje con M = Cd mod n.
  • Confidencialidad: Solo Bob, con su clave privada, puede descifrar el mensaje, asegurando que solo él pueda leer el contenido.

ESQUEMA 7: RSA (CONFIDENCIALIDAD)

  • 1️⃣ GÉNESIS 🗝️ Bob genera (e,n) Pública y (d,n) Privada.
  • 2️⃣ CIFRADO 🔒 Alice usa la Pública de Bob. Aplica la fórmula: C = M^e mod n.
  • 3️⃣ TRANSMISIÓN 📡 Alice envía el Criptograma (C).
  • 4️⃣ DESCIFRADO 🔓 Bob usa su Privada exclusiva. Aplica la fórmula: M = C^d mod n.

🛡️ Seguridad: Solo Bob posee 'd', la clave privada necesaria para recuperar M.

✍️

Caso Práctico 8: Firma Digital (Autenticidad RSA)

Solución Oficial

Enunciado: Bob quiere verificar que un mensaje recibido realmente proviene de Alice. Describe cómo Alice puede firmar digitalmente su mensaje usando RSA para asegurar la autenticidad.

Respuesta de la Plataforma:

  • Generación de Claves por Alice: Alice genera su par de claves RSA, obteniendo una clave pública (Alice_e, Alice_n) y una clave privada (Alice_d, Alice_n).
  • Firma Digital: Alice calcula un hash del mensaje H(M) usando una función de hash segura como SHA-256. Firma este hash usando su clave privada: S = H(M)Alice_d mod Alice_n.
  • Transmisión: Alice envía el mensaje M junto con la firma digital S a Bob.
  • Verificación por Bob:
    • Bob calcula el hash del mensaje recibido, H(M').
    • Usa la clave pública de Alice para verificar la firma: H(M) = SAlice_e mod Alice_n.
    • Si H(M') coincide con el hash calculado H(M), Bob puede confiar en que el mensaje proviene de Alice y no ha sido alterado.
  • Seguridad: La autenticidad se asegura porque solo Alice tiene acceso a su clave privada. Si la firma es válida, el mensaje no ha sido modificado y su origen es auténtico.

ESQUEMA 8: FIRMA DIGITAL (AUTENTICIDAD)

  • 1️⃣ HASHING ⚙️ Alice calcula el resumen H(M) con SHA-256.
  • 2️⃣ FIRMA ✍️ Alice cifra H(M) con SU clave Privada (d). Resulta S = H(M)^d mod n.
  • 3️⃣ TRANSMISIÓN 📤 Alice envía el paquete: [Mensaje (M) + Firma (S)].
  • 4️⃣ VERIFICACIÓN 🕵️‍♂️ Bob usa la Pública de Alice sobre (S) para extraer el Hash original, y lo compara con el Hash del mensaje (M) recibido.

✅ Autenticidad: Si la firma cuadra, solo pudo crearla Alice (la única poseedora de su Privada).

6. Glosario Técnico

Infraestructura de Clave Pública (PKI)

Marco tecnológico, políticas y servicios (como las Autoridades de Certificación) requeridos para gestionar, emitir y revocar los pares de claves asimétricas de forma escalable y confiable.

Algoritmo de Euclides Extendido

Ecuación matemática utilizada en la fase de generación de claves RSA para hallar el inverso multiplicativo modular, que permite obtener el exponente privado 'd'.

Firma Digital

Mecanismo criptográfico resultante de aplicar la clave privada de un emisor sobre el hash de un documento, garantizando Autenticidad, Integridad y No Repudio.

Módulo 'n'

En RSA, es el producto de dos números primos enormes (p y q). Se utiliza en la aritmética modular tanto para la operación de cifrado como para la de descifrado.

8. Flujo Operativo Híbrido Completo (E2E)

El estándar moderno (PGP/GPG) combina las Unidades 3 y 4: Firma Asimétrica + Cifrado Asimétrico para la clave de sesión + Cifrado Simétrico Bulk.

✍️

1. Firma (Integridad)

Alice aplica SHA-256 al documento y lo cifra con su Clave Privada.

➔
🔒

2. Cifrado (Confidencial)

Alice cifra el Doc+Firma usando una clave simétrica, y protege esa clave con la Pública de Bob.

➔
🔓

3. Descifrado

Bob usa su Privada para obtener la clave simétrica y descifrar el paquete original.

➔
✅

4. Verificación

Bob extrae la firma, usa la Pública de Alice y verifica los Hashes (No Repudio).
