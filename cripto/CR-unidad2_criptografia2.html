Unidad 2: Fundamentos de Criptografía

Soluciones Oficiales: Casos Prácticos 3 (DES) & 4 (RSA)

1. Fundamentos Matemáticos y Evolución

Cifrado Simétrico: DES

El Data Encryption Standard (DES) se basa en la estructura de redes de Feistel. Opera con bloques de 64 bits usando una clave efectiva de 56 bits.

  • Base Matemática: Permutaciones, sustituciones (S-boxes) y funciones XOR.
  • Evolución: Desarrollado en los 70s. Hoy es vulnerable a ataques de fuerza bruta (clave demasiado corta).
  • Vulnerabilidad: La búsqueda exhaustiva de claves es trivial con hardware moderno.

Cifrado Asimétrico: RSA

El algoritmo RSA se fundamenta en la dificultad computacional de factorizar números enteros grandes en sus primos componentes.

  • Base Matemática: Teorema de Euler, aritmética modular ($a^b \pmod n$), números primos.
  • Evolución: Propuesto en 1977. Estándar actual para intercambio de claves y firmas digitales.
  • Seguridad: Seguro mientras $n = p \times q$ sea suficientemente grande (>2048 bits).

2. Implementación Algorítmica y Despliegue

# Laboratorio: Cifrado Simétrico (Python + PyCryptodome)
from Crypto.Cipher import DES
import base64

# 1. Generación de Clave (Simulada)
key = b'SECRETA8' # 8 bytes = 64 bits (56 efectivos)

# 2. Preparación del Mensaje (Padding PKCS#5)
message = b"Alice necesita enviar un mensaje secreto."
padding_length = DES.block_size - len(message) % DES.block_size
message += bytes([padding_length]) * padding_length

# 3. Cifrado
cipher = DES.new(key, DES.MODE_ECB)
ciphertext = cipher.encrypt(message)
print(f"Texto Cifrado (Hex): {ciphertext.hex()}")

# 4. Descifrado (Bob usa la misma clave)
decipher = DES.new(key, DES.MODE_ECB)
plaintext = decipher.decrypt(ciphertext).rstrip(b'\x00')
print(f"Mensaje Original: {plaintext.decode()}")
Nota de Seguridad: En producción, DES debe ser reemplazado por AES-256 debido a su corta clave.
# Laboratorio: Cifrado Asimétrico (Lógica RSA Simplificada)
import random

# 1. Selección de Primos Pequeños (Para demostración)
p, q = 61, 53
n = p * q          # 3233
phi = (p-1)*(q-1)  # 3120
e = 17             # Exponente público (coprimo con phi)
# d = inverso(e, phi) -> Calculado manualmente o con Euclides Expandido
d = 2753           # Exponente privado

# 2. Cifrado por Alice (M = 65, ASCII 'A')
M = 65
C = pow(M, e, n)   # 65^17 mod 3233
print(f"Cifrado: {C}")

# 3. Descifrado por Bob
M_recuperado = pow(C, d, n) # C^2753 mod 3233
print(f"Descifrado: {M_recuperado} -> {chr(M_recuperado)}")
Parámetros Críticos: $p$ y $q$ deben ser grandes y aleatorios. $n$ se hace público, $d$ se mantiene secreto.

3. Triada CID Aplicada

Confidencialidad

Garantizada por la imposibilidad de revertir el algoritmo sin la clave. En RSA, solo quien tiene la privada puede leer. En DES, depende de que nadie robe la clave compartida.

Integridad

Aunque el cifrado protege el contenido, no asegura por sí solo que no fue alterado. Se requiere complementar con Hashes (SHA-256) o Firmas Digitales para validar integridad.

Disponibilidad

Balance seguridad vs. rendimiento. RSA es computacionalmente costoso (lento para grandes datos), por lo que se suele usar para intercambiar claves, mientras AES/DES cifra el flujo masivo.

4. Tarjetas de Memorización

Clave Privada vs Pública

Pública: Se comparte libremente (Cifrar/Verificar).
Privada: Secreto absoluto (Descifrar/Firmar).

Vector de Iniciación (IV)

Valor aleatorio usado en modo CBC para asegurar que idénticos mensajes produzcan diferentes textos cifrados.

Salting

Dato aleatorio añadido a una contraseña antes de calcular el hash para evitar ataques de diccionario/rainbow tables.

Teorema de Fermat

Base matemática para pruebas de primalidad y parte fundamental en la generación de claves RSA.

5. Casos Prácticos Oficiales

Caso Práctico 3: Cifrado Simétrico (DES)

Cuadrícula de Requisitos

Entrada Mensaje Plano + Clave Secreta (56 bits)
Algoritmo DES (Feistel Network, 16 rondas)
Proceso Dividir bloque 64b → Permutar → S-Boxes → Invertir
Salida Texto Cifrado (Hexadecimal)

Flujo Operativo (Alice & Bob)

1

Acuerdo Previo: Alice y Bob comparten la clave secreta por un canal seguro (fuera del sistema).

2

Cifrado: Alice divide su mensaje en bloques de 64 bits y aplica DES con la clave.

3

Transmisión: Envío del texto cifrado por red insegura.

4

Descifrado: Bob aplica la función inversa de DES con la misma clave para obtener el original.

⚠️ Punto de Fallo: El intercambio inicial de la clave secreta. Si es interceptado, todo el sistema colapsa.

Ficha de Verificación Técnica

Dificultad Computacional: Con una clave de 56 bits, el espacio de claves es $2^{56} \approx 7.2 \times 10^{16}$. Aunque parece grande, máquinas modernas pueden probar todas las claves en horas/días (ataque de fuerza bruta). Por ello, DES se considera obsoleto para protección de datos sensibles a largo plazo. Se recomienda migrar a AES-256.

Caso Práctico 4: Cifrado Asimétrico (RSA)

Cuadrícula de Requisitos

Entrada Mensaje M (numérico), Clave Pública (e, n)
Algoritmo RSA (Exponenciación Modular)
Proceso C = M^e mod n → Transmisión → M = C^d mod n
Salida Texto Cifrado C recuperado en M

Flujo Operativo (Alice & Bob)

1

Generación: Bob genera par $(e, n)$ pública y $(d, n)$ privada. Publica $(e, n)$.

2

Cifrado: Alice codifica su mensaje $M$ y calcula $C = M^e \pmod n$.

3

Transmisión: Envío del valor numérico $C$ por cualquier canal.

4

Descifrado: Bob aplica su clave privada: $M = C^d \pmod n$.

✅ Garantía: Incluso si alguien roba la clave pública, sin la privada $d$, es matemáticamente inviable recuperar $M$ (problema de factorización).

Ficha de Verificación Técnica

Ingeniería Inversa: Para romper RSA, un atacante necesitaría factorizar $n$ para encontrar $p$ y $q$, y así calcular $\phi(n)$ y luego $d$. Con claves de 2048 bits actuales, esto tomaría miles de años con supercomputadoras actuales. La seguridad reside en la asimetría computacional: multiplicar primos es fácil; factorizar el resultado es difícil.

6. Glosario Técnico

Cifrado Simétrico Mismo algoritmo y clave para cifrar/descifrar (ej. DES, AES).
Cifrado Asimétrico Par de claves pública/privada (ej. RSA, ECC).
Nonce Número usado una sola vez en protocolos criptográficos para prevenir replay attacks.
Padding Relleno añadido al mensaje para cumplir el tamaño de bloque requerido.
Entropía Medida de aleatoriedad esencial para generar claves seguras.
Key Exchange Protocolo para intercambiar claves seguras sobre canales públicos (ej. Diffie-Hellman).

8. Flujo Operativo (E2E)

1

Génesis de Clave

Generación de Números Primos (RSA) o acuerdo de secreto (DES).

2

Parametrización

Definición de N, E, D (RSA) o Padding/IV (DES).

3

Aplicación Algoritmo

Cifrado ($C=M^e \pmod n$) o Transformación de Bloques.

4

Transmisión Segura

Envío de datos cifrados por canal público.

5

Verificación/Descifrado

Uso de Clave Privada para recuperar M.

⚠️ Punto Crítico de Seguridad: La gestión y custodia de la Clave Privada en Fase 5.
