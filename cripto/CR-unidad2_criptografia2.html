APARTADO 1

1. Fundamentos Matemáticos y Evolución del Algoritmo

Estructuras Matemáticas Base

La seguridad de los sistemas criptográficos modernos se divide según su estructura operacional:

  • Asimétrica (RSA): Descansa directamente sobre la teoría de números y la complejidad computacional del problema de la factorización de enteros. Trabaja con aritmética modular sobre el producto de dos números primos grandes de alta entropía (n = p * q).
  • Simétrica (AES / DES): No depende de problemas matemáticos de factorización, sino de funciones de confusión y difusión implementadas mediante permutaciones y cajas de sustitución (S-Boxes) que alteran las estructuras de bits en bloques fijos.
Evolución Histórica
  • Clásicos (César, Vigenère): Sustitución simple. Rotos mediante análisis de frecuencias.
  • DES (Data Encryption Standard): Bloques de 64 bits y clave de 56 bits. Inseguro hoy frente a ataques de fuerza bruta.
  • AES (Advanced Encryption Standard): Matrices de estado y claves de 128/192/256 bits. Matemáticamente robusto.
  • RSA: Pilar del PKI. Seguro con claves ≥2048 bits, pero amenazado teóricamente por la futura computación cuántica (Algoritmo de Shor).
APARTADO 2

2. Implementación Algorítmica y Despliegue Técnico

Matemática Subyacente: Cálculo RSA

  1. Seleccionar dos números primos grandes aleatorios, p y q.
  2. Calcular módulo compuesto: n = p * q.
  3. Calcular función indicatriz de Euler: φ(n) = (p - 1) * (q - 1).
  4. Elegir exponente público e, coprimo con φ(n), tal que MCD(e, φ(n)) = 1.
  5. Calcular exponente privado d mediante inverso multiplicativo modular: (d * e) ≡ 1 (mod φ(n)) usando Euclides Extendido.

Comandos de Laboratorio (OpenSSL)

# 1. Generar clave privada RSA (2048 bits)
openssl genrsa -out privada.pem 2048

# 2. Extraer clave pública correspondiente
openssl rsa -in privada.pem -pubout -out publica.pem

# 3. Cifrar simétricamente con AES-256-CBC y Salt
openssl enc -aes-256-cbc -salt -in datos.txt -out cifrado.enc

# 4. Generar Hash SHA-256 para validación de integridad
openssl dgst -sha256 reporte.pdf
APARTADO 3

3. Fichas de Repaso (Tarjetas de Memorización)

Clave Pública vs Privada
Definición: Par asimétrico vinculado. La pública cifra/verifica; la privada descifra/firma.

Ejemplo: Compartir la pública para recibir archivos confidenciales.
Salting (Salado)
Definición: Bits aleatorios añadidos a una contraseña antes del hash.

Ejemplo: Mitiga ataques de Rainbow Tables.
Nonce
Definición: Número utilizado una única vez en un protocolo.

Ejemplo: Evita ataques de repetición (Replay).
Vector de Inicialización (IV)
Definición: Bloque de datos aleatorios para el primer bloque cifrado (CBC).

Ejemplo: Cifrar dos frases idénticas da resultados diferentes.
APARTADO 4

4. Casos Prácticos: Soluciones Oficiales Integradas

Caso 3: Cifrado Simétrico (DES)

Contexto: Alice y Bob desean comunicarse de manera segura usando DES. Describe el proceso y manejo de clave garantizando confidencialidad.

Solución Oficial:

  • Generación de Clave: Acuerdan de forma segura una clave secreta de 56 bits efectivos (64 bits total, 8 de paridad).
  • Cifrado (Alice): Divide el texto plano en bloques de 64 bits. Aplica DES a cada bloque usando la clave secreta (permutaciones, sustituciones y funciones de Feistel). Envía texto cifrado.
  • Descifrado (Bob): Usa la misma clave secreta. Al ser simétrico, aplica proceso inverso para recuperar el original.
  • Confidencialidad: Depende de mantener la clave secreta. (Nota: DES es inseguro hoy por su clave corta).
Caso 4: Cifrado Asimétrico (RSA)

Contexto: Alice quiere enviar un mensaje a Bob garantizando confidencialidad con RSA. Describe el cifrado y descifrado.

Solución Oficial:

  • Generación (Bob): Bob genera clave pública (e, n) y privada (d, n). Distribuye la pública.
  • Cifrado (Alice): Obtiene la pública de Bob. Convierte mensaje a número M. Cifra con la fórmula:
    C = Me mod n. Envía C.
  • Descifrado (Bob): Recibe el criptograma C y aplica su clave privada con la fórmula:
    M = Cd mod n.
  • Confidencialidad: Solo Bob puede descifrarlo porque solo él posee el exponente privado d.
APARTADO 5

5. Glosario Técnico

Cifrado Simétrico / Asimétrico
Simétrico: Única clave compartida de alta velocidad (AES). Asimétrico: Par de claves (pública/privada) para intercambio seguro o firmas (RSA).
Colisión Hash
Fallo donde dos entradas distintas producen el mismo hash de salida (ej. debilidad de MD5). Rompe el principio de integridad.
Entropía
Grado de aleatoriedad o imprevisibilidad. Esencial en la generación de claves para evitar patrones explotables.
Padding (Relleno)
Datos añadidos al final de un mensaje para que su tamaño sea múltiplo exacto del tamaño de bloque requerido por el algoritmo.
Key Exchange (Intercambio de Claves)
Mecanismo (ej. Diffie-Hellman) para acordar contraseñas simétricas sobre canales expuestos.
