🧠 CHULETA EXTENDIDA: PASO A PASO

Guía de resolución lógica para los 24 Casos Prácticos

🔐 Criptografía (Casos 1-6)

Caso 1: Primalidad (Miller-Rabin)

CP1
Paso 1: Descomposición
Expresa n-1 = d · 2^s. Divide por 2 hasta que quede impar (d).
Paso 2: Base Aleatoria
Elige un número a tal que 1 < a < n-1.
Paso 3: Cálculo Inicial
Calcula x = a^d mod n.
Paso 4: Iteración
Si x ≠ 1 y x ≠ n-1, cuadrar (x²) repetidamente s veces. Si nunca llega a n-1 → COMPUESTO.

Caso 2: Factorización (Euclides)

CP2
Paso 1: Entender el objetivo
Necesitamos el inverso modular d tal que e·d ≡ 1 (mod φ).
Paso 2: Algoritmo Euclides
Aplica el Algoritmo de Euclides Extendido a (e, φ) para encontrar coeficientes.
Paso 3: Obtener Inverso
El coeficiente resultante correspondiente a e es la clave privada d.
Paso 4: Verificación
Comprobar que (e × d) % φ == 1.

Caso 3: Simétrico (DES)

CP3
Paso 1: Acuerdo Previa
Alice y Bob comparten una única clave secreta (56 bits efectivos).
Paso 2: Bloqueo
Alice divide el mensaje en bloques fijos de 64 bits.
Paso 3: Procesamiento Feistel
Aplica 16 rondas de permutaciones, sustituciones y XOR con subclaves derivadas de la principal.
Paso 4: Descifrado Inverso
Bob usa la misma clave pero aplica las operaciones en orden inverso para recuperar el texto.

Caso 4: Asimétrico (RSA)

CP4
Paso 1: Generación (Bob)
Genera primos p, q. Calcula n=p·q y φ=(p-1)(q-1). Elige e y calcula d.
Paso 2: Distribución
Bob publica (e, n) (Clave Pública). Oculta d (Clave Privada).
Paso 3: Cifrado (Alice)
Convierte mensaje M a número. Calcula C = M^e mod n usando la pública.
Paso 4: Descifrado
Bob recibe C. Calcula M = C^d mod n con su privada. Solo él puede hacerlo.

Caso 5: Moderno (AES-CBC)

CP5
Paso 1: Selección Clave
Acuerdan clave larga (128/256 bits). Más segura que DES.
Paso 2: Vector IV
Alice genera un IV aleatorio único para esta comunicación.
Paso 3: Encadenamiento (CBC)
Primero: Bloque1 XOR IV. Luego: cifrar. Segundo: Bloque2 XOR SalidaAnterior. Y así sucesivamente.
Paso 4: Transmisión
Envía [IV || BloquesCifrados]. Bob descifra invirtiendo el proceso (XOR con salida anterior).

Caso 6: Integridad (SHA-256)

CP6
Paso 1: Hash Origen
Alice calcula H_origen = SHA256(Mensaje_original).
Paso 2: Empaquetado
Envía el paquete: [Mensaje + H_origen].
Paso 3: Recepción
Bob recibe el paquete. Separa el mensaje del hash recibido.
Paso 4: Comparación
Bob recalcula H_destino = SHA256(Mensaje_recibido). ¿H_origen == H_destino? Sí = Íntegro.

🔗 Blockchain (Casos 7-14)

Caso 7: Estructura de Bloque

CP7
Paso 1: Datos
Recopilar transacciones pendientes (Tx) en memoria.
Paso 2: Vinculación
Incluir en el cabecera el Hash del bloque anterior (cadena).
Paso 3: Timestamp & Nonce
Añadir hora actual y buscar un Nonce aleatorio válido.
Paso 4: Hash Final
Calcular SHA256(header) que cumpla la dificultad. Este es el identificador del bloque.

Caso 8: Minería PoW

CP8
Paso 1: Objetivo
Encontrar un Hash < Target (Dificultad). Requiere prueba de trabajo.
Paso 2: Búsqueda
Mineros prueban trillones de Nonces aleatorios restando al header.
Paso 3: Hallazgo
Un minero encuentra el nonce correcto y lo anuncia a la red.
Paso 4: Validación
Otros nodos verifican rápido el hash. Si es válido, añaden el bloque y dan recompensa.

Caso 9: Consenso PoS

CP9
Paso 1: Apuesta (Staking)
Validadores bloquean monedas (ej. 32 ETH) como garantía.
Paso 2: Selección
Protocolo elige aleatoriamente un validador basado en su apuesta.
Paso 3: Propuesta
El elegido crea el bloque y lo propone.
Paso 4: Attestation
Otro grupo valida el bloque. Si actúa mal, se le penaliza (Slashing).

Caso 10: Smart Contracts

CP10
Paso 1: Codificación
Escribir lógica "If X entonces Y" en lenguaje específico (Solidity).
Paso 2: Despliegue
Subir el código a la blockchain. Se asigna una dirección única.
Paso 3: Espera
El contrato espera ser "disparado" por una transacción externa.
Paso 4: Ejecución
Se ejecuta automáticamente sin intermediarios. El estado cambia de forma inmutable.

Caso 11: Consenso BFT

CP11
Paso 1: Propuesta
Un líder propone un bloque o valor.
Paso 2: Votación Preliminar
Nodos reciben y votan "Pre-prepare".
Paso 3: Votación Formal
Nodos intercambian votos ("Prepare") entre sí.
Paso 4: Confirmación
Si 2/3 de los votos coinciden, el bloque se considera final (tolerancia fallos bizantinos).

Caso 12: Escalabilidad (L2)

CP12
Paso 1: Canales Off-chain
Usuarios abren canal de pagos fuera de la cadena principal.
Paso 2: Transacciones Rápidas
Intercambian firmas digitales múltiples veces instantáneamente.
Paso 3: Cierre
Cuando terminan, cierran el canal.
Paso 4: Settlement
Solo el resultado final se graba en la cadena principal (On-chain), ahorrando gas.

Caso 13: Forks

CP13
Paso 1: Actualización
Propuesta de cambio en las reglas del protocolo.
Paso 2: Soft Fork
Reglas más restrictivas. Compatible hacia atrás. No divide red.
Paso 3: Hard Fork
Reglas incompatibles. Dividen la historia en dos cadenas separadas.
Paso 4: Comunidad
La comunidad decide qué cadena seguir, creando dos activos (ej. BTC y BCH).

Caso 14: Ataque 51%

CP14
Paso 1: Control
Un entidad obtiene >51% del poder de hash (minería) o stake.
Paso 2: Doble Gasto
Puede revertir sus propias transacciones gastando dos veces las mismas monedas.
Paso 3: Exclusión
Puede impedir confirmación de otros bloques.
Paso 4: Realidad
Es extremadamente costoso en redes grandes (Bitcoin), por eso es raro.

📊 Learning Analytics (Casos 15-24)

Caso 15: Proceso KDD

CP15
Paso 1: Selección
Identificar objetivos y seleccionar datos relevantes.
Paso 2: Preprocesado
Limpiar datos, tratar valores nulos y ruido.
Paso 3: Transformación
Normalizar o reducir dimensionalidad de datos.
Paso 4: Mining
Aplicar algoritmos (Cluster/Tree) para extraer patrones.
Paso 5: Evaluación
Interpretar resultados y validar si cumplen el objetivo.

Caso 16: Clustering (K-Means)

CP16
Paso 1: Elección K
Decidir número de grupos (centroides) iniciales.
Paso 2: Asignación
Asignar cada punto al centroide más cercano (distancia euclídea).
Paso 3: Actualización
Recalcular la posición de los centroides como media de los puntos asignados.
Paso 4: Convergencia
Repetir pasos 2 y 3 hasta que los centroides ya no se muevan.

Caso 17: Clasificación (Árbol)

CP17
Paso 1: Raíz
Elegir atributo que mejor separa clases (Information Gain).
Paso 2: División
Crear ramas según los valores posibles del atributo.
Paso 3: Recursividad
Repetir el proceso en cada sub-nodo hasta pureza.
Paso 4: Hojas
Las hojas finales representan la clase predicha (Sí/No, Aprobo/Reprobó).

Caso 18: Regresión Lineal

CP18
Paso 1: Datos
Recopilar pares (X, Y) ej. Horas estudio vs Nota.
Paso 2: Modelo
Asumir relación lineal: Y = mX + b.
Paso 3: Ajuste
Usar mínimos cuadrados para hallar m y b que minimicen el error.
Paso 4: Predicción
Insertar nuevo X en la fórmula para predecir Y.

Caso 19: Asociación (Apriori)

CP19
Paso 1: Frecuente Items
Contar apariciones de cada ítem individualmente.
Paso 2: Filtrado Support
Descartar ítems que no superan el mínimo soporte (%).
Paso 3: Combinaciones
Generar pares/tríos de ítems frecuentes y calcular su soporte.
Paso 4: Reglas
Calcular confianza: P(A|B). Generar regla fuerte: A → B.

Caso 20: Visualización

CP20
Paso 1: Métrica KPI
Definir qué dato es crucial (ej. Tasa de abandono).
Paso 2: Tipo Gráfico
Elegir según dato: Barras (comparación), Línea (tendencia), Pastel (partes).
Paso 3: Diseño Limpio
Evitar colores innecesarios, usar etiquetas claras.
Paso 4: Interactividad
Añadir filtros para permitir "drill-down" al usuario.

Caso 19: Datos y Correlación

CP19
Paso 1:
📂 **Pandas**: Cargar dataset desde CSV.
Paso 2:
📊 **Seaborn**: Crear gráfico de dispersión (Horas vs. Notas).
Paso 3:
📈 **Tendencia**: Añadir línea de correlación y detectar outliers.
Paso 4:
💾 **Exportar**: Guardar imagen o incluir en informe.

Caso 20: Dashboard Power BI

CP20
Paso 1:
📥 **Importar**: Conectar datos de Excel/Moodle a Power BI.
Paso 2:
📊 **Visualizar**: Gráficos de barras (participación) y líneas (tendencias).
Paso 3:
🏆 **KPIs**: Tarjeta con promedio general y filtros por materia/profe.
Paso 4:
🌐 **Accesibilidad**: Colores aptos y publicar dashboard online.

Caso 21: Riesgo Académico

CP21
Paso 1: Evaluar Reglas
Aplicar condiciones a cada alumno:
• A01: accesos≤2 → SÍ (Riesgo)
• A02: entregas≥40 → NO
• A03: accesos≤2 → SÍ
• A04: entregas≥40 → NO
• A05: entregas<40 → SÍ
Paso 2: Ordenar Prioridad
Ordenar solo los "SÍ" (Riesgo) por menor % entregas:
1. A03 (0%)
2. A01 (20%)
3. A05 (30%)
Luego los "NO" (Mantener ritmo).
Paso 3: Definir Acción
• Grupo Riesgo (A03/A01/A05): Enviar mensaje personal + recordatorio entrega.
• Grupo Seguro: Feedback positivo.
Resultado Final
Lista priorizada de intervención docente basada en severidad.

Caso 22: Predicción Regla

CP22
Paso 1: Aplicar Umbral
Regla: Si Notas<50 → Predicción 1 (Riesgo), sino verificar ausencias.
Paso 2: Evaluar Casos
• S01 (30<50) → Pred 1 (Real 1) ✅
• S02 (45<50) → Pred 1 (Real 1) ✅
• S03 (80≥50, Aus=1) → Pred 0 (Real 0) ✅
• S05 (20<50) → Pred 1 (Real 1) ✅
Paso 3: Validar Resto
• S04, S06, S08 (Notas altas, ausencias bajas) → Pred 0 (Correcto).
• Total Aciertos: 8 de 8 (100%).
Conclusión
La regla simple funciona perfectamente en este conjunto pequeño y limpio.

CP23

Caso 24: Dashboard Simple

CP24
Paso 1: Gráficos Básicos
Usar barras (cantidad), dispersión (relación), líneas (tiempo). Evitar complejidad 3D.
Paso 2: Claridad
Títulos descriptivos ("Rendimiento Promedio"), leyendas explícitas y unidades claras.
Paso 3: Semántica Visual
Colores: Verde (Mejora/OK), Rojo/Naranja (Riesgo). Iconos para resaltar alertas.
Paso 4: Interpretación
Añadir texto explicativo debajo de cada gráfico. El usuario debe entenderlo "de un vistazo" sin conocimientos técnicos.
