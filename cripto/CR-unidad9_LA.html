1. Introducción al Aprendizaje Automático

En esta unidad profundizamos en los algoritmos que permiten a las máquinas "aprender" de los datos. Distinguimos claramente entre dos grandes familias:

Aprendizaje Supervisado

El modelo aprende a partir de ejemplos etiquetados (entradas + respuestas correctas).

  • Objetivo: Predecir una etiqueta o valor continuo.
  • Ejemplos: Clasificación (Spam/No Spam), Regresión (Previsión de ventas).
  • Algoritmos: Árboles de Decisión, SVM, Regresión Logística, K-NN.

Aprendizaje No Supervisado

El modelo explora datos sin etiquetas para encontrar patrones ocultos.

  • Objetivo: Descubrir estructura interna (grupos, reducción dimensiones).
  • Ejemplos: Segmentación de clientes, Compresión de datos.
  • Algoritmos: K-Means, DBSCAN, PCA, Reglas de Asociación.

2. Algoritmos y Funcionamiento Interno

Clasificación: Árboles de Decisión

Modelo basado en reglas "Si-Entonces". Divide el espacio de características en regiones cada vez más puras.

Nodo Raíz
Sí
No

Clustering: K-Means

Agrupamiento iterativo. Selecciona K centroides iniciales, asigna puntos al centroide más cercano y recalculate la posición de los centroides hasta converger.

SVM (Máquinas de Vectores de Soporte)

Busca el hiperplano óptimo que separa clases maximizando el margen entre ellas. Es muy efectivo en espacios de alta dimensionalidad.

3. Métricas y Evaluación de Modelos

Matriz de Confusión

Predicha \ RealPositivoNegativo
PositivoVP (True Pos)FP (False Pos)
NegativoFN (False Neg)VN (True Neg)

Precisión: VP / (VP+FP) | Recall: VP / (VP+FN)

Regresión

  • RMSE (Root Mean Squared Error): Mide el error promedio en las unidades de la variable objetivo. Menor es mejor.
  • R² (Coeficiente de Determinación): Porcentaje de varianza explicada por el modelo (0 a 1). Más cerca de 1 es mejor ajuste.

Caso Práctico 17: Predicción Académica (Regresión)

Solución Oficial

Una universidad desea predecir las calificaciones finales de sus estudiantes en una asignatura de programación, utilizando datos como número de horas de estudio, asistencia a clases y participación en foros virtuales. ¿Cómo puede aplicarse un modelo de regresión supervisada?

Respuesta de la Plataforma:

  • Recolección de Datos: Variables independientes (horas estudio, participación, entregas, asistencia) y Variable dependiente (calificación final).
  • Preprocesamiento: Normalizar valores numéricos y tratar valores faltantes (ej: imputar media en asistencia).
  • Modelo: Aplicar Regresión Lineal Múltiple. Dividir dataset en entrenamiento (80%) y prueba (20%).
  • Evaluación: Utilizar métricas R² y RMSE para medir precisión.
  • Aplicación: Predecir calificaciones antes del examen para ofrecer tutorías personalizadas.
  • 📊
    RECOLECCIÓN Y PREPARACIÓN: Definir variables predictoras (horas, asistencia) y objetivo (nota).
  • 🧮
    ENTRENAMIENTO DEL MODELO: Ajustar coeficientes de la Regresión Lineal Múltiple para minimizar el error cuadrático.
  • 📉
    VALIDACIÓN ESTADÍSTICA: Comprobar R² y RMSE en conjunto de prueba para asegurar generalización.
  • 🚀
    INTERVENCIÓN PEDAGÓGICA: Usar predicciones bajas para activar alertas de apoyo estudiantil.
⚠️

Caso Práctico 18: Detección de Abandono (Árboles)

Solución Oficial

Un centro educativo en línea quiere identificar a los estudiantes con riesgo de abandonar sus cursos. Dispone de datos como número de accesos a la plataforma, tiempo medio de conexión y cumplimiento de tareas. ¿Cómo puede un árbol de decisión ayudar?

Respuesta de la Plataforma:

  • Datos de Entrada: Accesos semanales, tiempo conexión, % tareas entregadas, participación foros.
  • Modelo: Crear Árbol de Decisión clasificando en "Activo" (1) o "En Riesgo" (0).
  • Entrenamiento: Dividir datos y evaluar con Precisión, Recall y Matriz de Confusión.
  • Interpretación: El árbol genera reglas claras. Ejemplo: "Si tiempo conexão < 2h/semana Y tareas < 50% → Riesgo de abandono".
  • Beneficios: Detección temprana, mejora retención, intervención personalizada.
  • 🌳
    CONSTRUCCIÓN DEL ÁRBOL: Selección de la mejor variable split (ej: Tareas Entregadas) basada en impureza (Gini).
  • 🔀
    GENERACIÓN DE REGLAS: Recorrido desde raíz hasta hoja para obtener caminos de decisión (SI Acceso bajo ENTONCES Riesgo).
  • ✅
    VALIDACIÓN CRUZADA: Asegurar que el árbol no esté sobreajustado (Overfitting) a los datos históricos.
  • 📧
    ACCIÓN AUTOMÁTICA: Disparar emails de alerta a tutores basados en la clasificación final.

5. Fichas Rápidas (Conceptos Core)

🎯

Supervisado

Definición

Aprendizaje con etiquetas conocidas (Entrada X → Salida Y). Objetivo: Predecir.

🔍

No Supervisado

Definición

Sin etiquetas. El modelo descubre patrones o grupos ocultos en los datos.

📉

Overfitting

Riesgo

Cuando el modelo memoriza el ruido del entrenamiento y falla en datos nuevos.

⚖️

Hiperplano

SVM

Línea (o plano) que separa clases. SVM busca maximizar el margen alrededor de este.

🔄

Cross-Val

Validación

Dividir datos en K partes, rotando entren/prueba para evaluar robustez.

🌲

Gini

Impureza

Medida usada en árboles para decidir splits. Busca minimizarla (maximizar pureza).

📏

PCA

Reducción

Análisis Componentes Principales. Reduce dimensiones manteniendo máxima varianza.

🤝

K-NN

Vecinos

Clasifica basándose en la mayoría de votos de sus 'k' vecinos más cercanos.

6. Glosario Operativo

Label (Etiqueta)

La respuesta correcta que queremos predecir en aprendizaje supervisado (ej. "Sí/No", "Precio").

Feature (Característica)

Variable independiente utilizada como entrada para el modelo (ej. "Edad", "Horas estudio").

Cluster

Grupo de datos similares encontrados por algoritmos no supervisados como K-Means.

Support Vector

Puntos de datos que definen el margen en SVM. Son los más difíciles de clasificar.

Epoch

Una pasada completa a través de todo el conjunto de datos de entrenamiento durante el proceso de aprendizaje.

AUC-ROC

Área bajo la curva ROC. Medida de rendimiento para problemas de clasificación binaria.

8. Flujo Operativo E2E: Ciclo de Vida ML

El proceso estándar desde el problema de negocio hasta el modelo desplegado.

🏢

1. Definición del Problema

Negocio
▼
Traducir el problema ("Perdemos clientes") en un problema de minería de datos ("Predicción de abandono con etiquetas 0/1").
⬇
📊

2. Preparación de Datos

Ingeniería
▼
Limpieza (valores nulos), integración de fuentes, transformación (normalización) y selección de características relevantes.
⬇
🧠

3. Modelado

Algoritmo
▼
Selección de algoritmo (ej. K-Means o Árbol), entrenamiento con el conjunto Train y ajuste de hiperparámetros.
⬇
✅

4. Evaluación

Validación
▼
Prueba del modelo con datos nunca vistos (Test Set). Análisis de métricas (Precisión, Recall, R²). Si no aprueba, volver a paso 3.
⬇
🚀

5. Despliegue

Producción
▼
Integración del modelo en sistemas reales (API, Dashboard) y monitorización continua del rendimiento (Data Drift).
