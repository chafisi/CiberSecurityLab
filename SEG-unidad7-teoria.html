1. Capa de Aplicación y OWASP

La Capa 7 es crítica porque expone la lógica de negocio a Internet y a la interacción humana. Un firewall perimetral tradicional está ciego ante peticiones HTTP "legítimas" que contienen comandos destructivos.

La Tríada de Inyecciones y Manipulaciones

Ataque ¿Qué se rompe? Objetivo Principal Gravedad (Impacto)
SQLi (Inyección SQL) El Servidor / Base de Datos Robar, modificar o borrar datos masivos corporativos. Crítica (Máxima)
XSS (Cross-Site Scripting) El Usuario (Cliente/Navegador) Robar sesiones (cookies) o espiar inyectando JavaScript. Alta
CSRF (Cross-Site Request Forgery) La Acción (Lógica) Obligar al usuario autenticado a realizar transacciones falsas. Media/Alta

* Nota de auditoría: Mientras el WAF mitiga SQLi y XSS basándose en firmas, el CSRF requiere protecciones nativas en el código de la aplicación (Tokens Anti-CSRF).

2. Identidad, Sesiones y DevSecOps

Gestión de Identidad y Sesión

"No hackean el cifrado: roban credenciales o secuestran sesiones". Una cookie robada equivale a un *bypass* completo del login.

  • Autenticar (Quién eres): Fomentar MFA robusto (FIDO2) frente a OTPs por SMS.
  • Autorizar (Qué puedes hacer): Implementar controles RBAC estrictos.
  • Sesión (Estado): Prevenir Session Hijacking configurando banderas Secure y HttpOnly en cookies, forzando expiración y rotando el ID tras cada login.

DevSecOps: Seguridad en el Código

La seguridad no se aplica al final; se automatiza en el pipeline CI/CD (Continuous Integration / Continuous Deployment).

  • SAST (Static Application Security Testing): Analiza el código fuente ("caja blanca") antes de compilar buscando variables no saneadas.
  • DAST (Dynamic Application Security Testing): Ataca la aplicación en ejecución ("caja negra") simulando XSS/SQLi.
  • SCA (Software Composition Analysis): Inventaría dependencias de terceros para evitar librerías vulnerables heredadas.

3. Perspectiva de Auditoría

🎭

El Factor Humano (Phishing y Sesiones)

El perímetro más débil en la Capa 7 es el ser humano. Un auditor no solo revisa configuraciones, sino procesos. Si un ataque de Phishing logra evadir el MFA (ej. ataques AitM - Adversary in the Middle), se evidencia la necesidad de transicionar a estándares criptográficos ligados al hardware como WebAuthn/FIDO2, donde la clave privada nunca abandona el dispositivo del empleado.

⏳

Caducidad de Criptografía vs. Disponibilidad

Implementar TLS no termina al instalar el certificado. La auditoría exige herramientas automatizadas (como el protocolo ACME) y escaneos de monitorización continuos. Un certificado expirado no es solo una "alerta de navegador", es una interrupción directa (Caída de Disponibilidad) de servicios críticos, APIs y pasarelas de pago.

4. Fichas Rápidas

CSRF

Cross-Site Request Forgery. Ataque que fuerza al navegador de un usuario autenticado a ejecutar acciones no deseadas en una aplicación de confianza.

FIDO2 / WebAuthn

Estándar de MFA sin contraseñas basado en criptografía de clave pública. Inmune al phishing tradicional porque el "secreto" se queda en el hardware local.

ACME

Automated Certificate Management Environment. Protocolo utilizado para automatizar la solicitud, validación y renovación de certificados TLS/SSL (ej. Let's Encrypt).

HttpOnly

Bandera de seguridad configurada en una Cookie que impide que los scripts del lado del cliente (JavaScript) puedan leerla, mitigando drásticamente los ataques XSS.

6. Glosario Técnico

WAF (Web Application Firewall)

Firewall específico de Capa 7 diseñado para proteger servidores web interceptando peticiones maliciosas (XSS, SQLi) analizando el tráfico HTTP/HTTPS.

SAST / DAST

Mecanismos de DevSecOps. SAST revisa vulnerabilidades en el código fuente (caja blanca). DAST ataca la aplicación en ejecución (caja negra).

PKI (Public Key Infrastructure)

Marco tecnológico e institucional (Autoridades de Certificación) que gestiona la creación, almacenamiento y revocación de certificados digitales.

Token Anti-CSRF

Cadena alfanumérica única y aleatoria generada por el servidor e incrustada en formularios web. El servidor rechaza la petición si este token no coincide.
