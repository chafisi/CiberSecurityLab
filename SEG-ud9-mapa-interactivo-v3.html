Perspectiva Global: La Triada CID

La compresión va antes del cifrado. El cifrado aporta Confidencialidad. Las técnicas de codificación aportan Integridad y Disponibilidad. (Ver versión anterior para más detalles).

Nodos Verdes: Técnicas de Detección (Visual & Ataques)

CRC (División Polinómica)

Integridad

Calcula el resto matemático de dividir el mensaje completo por un polinomio predefinido. Muy usado en Ethernet e IoT.

Mensaje: 11010110 ÷ Polinomio = Firma: 011

⚠️ Vulnerabilidad: Modificación Intencionada

El CRC no usa claves criptográficas. Un atacante (Man-in-the-Middle) puede interceptar "Pagar 10€", cambiarlo a "Pagar 99€", y recalcular el CRC él mismo. El receptor lo validará como perfecto. Solución: Usar HMAC (Firmas Criptográficas), no solo CRC.

Bits de Paridad

Integridad

Añade un bit al final de un bloque para asegurar que la cantidad total de "unos" sea siempre un número par (o impar). Usado en RAM (ECC).

1 0 1 1 ➔ 3 Unos ➔ 1
Ahora hay 4 unos (Número Par ✅)

⚠️ Vulnerabilidad: Evasión por Múltiple Fallo

Solo detecta si cambia un bit. Si una interferencia fuerte (o un atacante) voltea dos bits a la vez, el resultado sigue siendo par. El sistema aceptará un archivo corrupto sin darse cuenta, comprometiendo la integridad silenciosamente.

Nodos Verdes: Técnicas de Corrección (FEC)

LDPC (Low-Density Parity-Check)

Disponibilidad Avanzada

Código avanzado usado en 5G y Wi-Fi 6. Funciona cruzando datos en una matriz dispersa (grafos bipartitos). Resuelve errores resolviendo ecuaciones de forma iterativa y probabilística.

Nodos Dato ⤡ ⤢ ⤧ ⤨ Nodos Chequeo

⚠️ Vulnerabilidad: Energy/Compute Exhaustion (DoS)

LDPC decodifica en iteraciones (cuanto más ruido, más ciclos de CPU necesita para adivinar el dato original).

Ataque Práctico: Un atacante genera ruido artificial constante (Jamming moderado). La señal no se cae, pero obliga al receptor (ej. un dron o sensor IoT) a usar el 100% de CPU para decodificar, drenando su batería en minutos y provocando una Denegación de Servicio física.

Reed-Solomon

Disponibilidad en Ráfaga

Opera agrupando bits en "símbolos" en lugar de bits individuales. Es el rey reparando errores contiguos masivos (ej. códigos QR manchados o arañazos en CDs).

D1 D2 💥 💥 FEC1 FEC2

⚠️ Vulnerabilidad: Límite Matemático Estricto

Si Reed-Solomon tiene "2 símbolos" de redundancia (FEC), puede recuperar como máximo "2 símbolos" destruidos. Si un ataque o interferencia destruye 3 símbolos, todo el bloque es matemáticamente irrecuperable. Los atacantes pueden calcular este margen para emitir pulsos electromagnéticos justos que superen el umbral sin gastar energía de más.

Códigos de Hamming

Integridad de Hardware

Mecanismo ultra-rápido de lógica computacional. Intercala bits de paridad en posiciones clave de los datos para triangular el bit exacto que falló.

P1 P2 D1 P3 D2 D3 D4

⚠️ Vulnerabilidad: Corrección Falsa (Corrupción Activa)

Hamming(7,4) detecta errores de 2 bits, pero solo corrige 1 bit. Si un atacante usa *Fault Injection* (ej. láser sobre un chip) para alterar 2 bits simultáneos, el algoritmo se confunde y "corrige" un tercer bit que estaba sano. El resultado es que el propio mecanismo de defensa corrompe aún más el sistema, provocando fallos impredecibles o escalada de privilegios.
